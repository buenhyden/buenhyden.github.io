URI (Uniform Resource Identifier)#

인터넷 상의 자원을 고유하게 식별하기 위한 문자열로 URL과 URN은 모두 URI의 하위 개념이라고 볼 수 있다.

기본구조:

1

scheme:[//authority]path[?query][#fragment]

대괄호([])로 표시된 부분은 선택적 요소이다.

  • scheme: 어떤 종류의 리소스를 식별하는지, 그리고 어떤 방식으로 접근해야 하는지를 나타낸다.
    특징:
    - 항상 알파벳으로 시작해야 합니다
    - 콜론(:)으로 다른 부분과 구분됩니다
    - 대소문자를 구분하지 않습니다
  • authority: 리소스를 관리하는 주체
    세부구성:
    - Userinfo (사용자 정보): 선택적 구성요소
    - 보안상의 이유로 현대 웹에서는 권장되지 않습니다
    - ‘@’ 기호로 호스트와 구분됩니다
    - Host (호스트): 필수 구성요소
    - 도메인 이름 또는 IP 주소가 올 수 있습니다
    - 리소스가 위치한 서버를 식별합니다
    - Port (포트): 선택적 구성요소
    - 콜론(:)으로 시작합니다
    - 생략 시 기본 포트가 사용됩니다 (http는 80, https는 443)
  • path: 리소스의 경로
    특징과 역할:
    - 슬래시(/)로 시작합니다
    - 계층적 구조를 나타냅니다
    - 여러 세그먼트로 구성될 수 있습니다
    - 각 세그먼트는 슬래시(/)로 구분됩니다
    - 리소스의 구체적인 위치나 식별자를 나타냅니다
  • query: 추가적인 매개변수
    쿼리의 특징과 사용:
    - 물음표(?)로 시작합니다
    - 키=값 형태의 매개변수들로 구성됩니다
    - 여러 매개변수는 앰퍼샌드(&)로 구분됩니다
    - 리소스를 필터링하거나 추가 정보를 전달하는 데 사용됩니다
    - URL 인코딩이 필요할 수 있습니다
  • fragment: 리소스 내의 특정 부분을 가리킴
    프래그먼트의 특징과 용도:
    - 해시(#) 기호로 시작합니다
    - 문서 내의 특정 부분을 가리킵니다
    - 서버로 전송되지 않고 클라이언트에서만 사용됩니다
    - HTML 문서에서 특정 섹션으로 이동하는 데 주로 사용됩니다

특징:

  1. 통일성(Uniform): 리소스 식별 방식이 일관됩니다.
  2. 리소스(Resource): 식별 가능한 모든 것을 대상으로 합니다.
  3. 식별자(Identifier): 리소스를 고유하게 식별합니다.

URL (Uniform Resource Locator)#

URI의 가장 일반적인 형태로, 인터넷상에서 자원의 위치를 나타내는 주소이다.
URL은 “어떻게” 자원에 접근할 수 있는지를 알려준다.

예시와 구성요소:

1

https://john:password@www.example.com:8080/blog/post/123?category=tech&status=public#comments
  • 프로토콜 (Scheme) 프로토콜은 URL의 맨 앞에 위치하며, 리소스에 접근하는 방법을 정의.
    https://
    주요 프로토콜들:
    - http: 일반적인 웹 통신
    - https: 보안이 적용된 웹 통신
    - ftp: 파일 전송
    - mailto: 이메일
    - file: 로컬 파일 시스템
  • 사용자 정보(Userinfo): 서버 인증에 필요한 사용자 이름과 비밀번호를 포함
    john:password@
    보안상의 이유로 현대의 웹에서는 잘 사용되지 않는다.
  • 호스트 (Host) 서버의 도메인 이름이나 IP 주소를 나타낸다.
    www.example.com
    서브도메인을 포함할 수 있으며, IP 주소를 직접 사용할 수도 있다.
  • 포트 (Port) 서버의 특정 포트 번호를 지정합니다.
    :8080
    기본 포트(http:80, https:443)를 사용할 경우 생략 가능
  • 경로 (Path) 서버에서 리소스의 위치를 나타냅니다.
    /blog/post/123 계층적 구조를 가지며, 각 단계는 슬래시(/)로 구분
  • 쿼리 문자열 (Query String) 서버에 전달할 추가 매개변수들을 포함합니다.
    ?category=tech&status=public
    물음표(?)로 시작
    키=값 형태로 구성
    여러 매개변수는 앰퍼샌드(&)로 구분
  • 프래그먼트 (Fragment) 페이지 내의 특정 섹션을 가리킵니다.
    #comments
    해시(#) 기호로 시작하며, 서버로 전송되지 않고 브라우저에서만 처리

URL 인코딩 (URL Encoding)#

URL에서 사용할 수 없는 문자들을 안전하게 전송하기 위한 방법
ASCII가 아닌 문자나 특수문자를 %와 16진수 코드로 변환

URL 인코딩이 필요한 경우:

  • 공백이나 특수문자 포함 시
  • 비ASCII 문자(한글 등) 포함 시
  • 예약된 문자를 데이터로 사용할 때

URL 인코딩 주의사항:

  • encodeURI()와 encodeURIComponent()의 차이점 이해

    1
2
3
4
5
6
7

    // encodeURI는 URL 전체를 인코딩할 때 사용
console.log(encodeURI('https://example.com/path with space'));
// 출력: https://example.com/path%20with%20space

// encodeURIComponent는 매개변수 값을 인코딩할 때 사용
console.log(encodeURIComponent('https://example.com/path'));
// 출력: https%3A%2F%2Fexample.com%2Fpath

  • 이미 인코딩된 문자열을 다시 인코딩하지 않도록 주의

  • 쿼리 문자열 생성 시 각 매개변수 값을 개별적으로 인코딩

자주 사용되는 인코딩 문자:

원본 문자인코딩설명
공백%20단어 사이의 공백을 나타냅니다
!%21느낌표
"%22큰따옴표
#%23해시 태그, 프래그먼트 식별자
$%24달러 기호
%%25퍼센트 기호
&%26앰퍼샌드, 쿼리 스트링 구분자
'%27작은따옴표
(%28왼쪽 괄호
)%29오른쪽 괄호
*%2A별표
+%2B더하기 기호
,%2C쉼표
/%2F슬래시, 경로 구분자
:%3A콜론
;%3B세미콜론
=%3D등호, 쿼리 스트링 할당 연산자
?%3F물음표, 쿼리 스트링 시작
@%40골뱅이, 이메일 주소 구분자
[%5B왼쪽 대괄호
]%5D오른쪽 대괄호

URN (Uniform Resource Name)#

자원의 이름을 지정하는 URI이다.
URN은 자원의 “무엇"인지를 식별한다.
URN은 자원의 위치가 변경되더라도 동일하게 유지된다.

특징:

  1. 영속성: 리소스의 위치가 변경되어도 동일한 URN으로 식별 가능합니다.
  2. 위치 독립성: 특정 위치에 종속되지 않습니다.
  3. 고유성: 각 리소스에 대해 유일한 이름을 제공합니다.
