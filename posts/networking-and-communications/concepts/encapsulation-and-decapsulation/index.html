encapsulation and decapsulation

Encapsulation (캡슐화) and Decapsulation (역캡슐화)#

  • 데이터에 헤더 (Header) 를 붙이고 아래 계층에 보내는 것을 캡슐화 (Encapsulation), 데이터에 헤더를 제거하고 위 계층에 보내는 것을 역캡슐화 (Decapsulation)
  • 데이터 전송시 캡슐화가 이뤄지고 수신시 역캡슐화 (Decapsulation) 가 이뤄진다.
  • 데이터에 헤더 (Header) 를 붙이고 아래 계층에 보내는 것을 캡슐화 (Encapsulation), 데이터에 헤더를 제거하고 위 계층에 보내는 것을 역캡슐화 (Decapsulation) 라고 한다.

중요성#

  1. 모듈성: 각 계층의 기능을 논리적으로 분리하여 네트워크 프로토콜의 모듈식 설계를 가능하게 한다.
  2. 추상화: 다른 통신 계층에서 전달되는 정보의 구조를 추상화한다.
  3. 효율성: 대량의 데이터를 효율적으로 전송할 수 있게 한다.
  4. 보안: 데이터의 안전한 전송을 보장하며, 암호화된 통신 채널을 구축하는 데 도움을 준다.
  5. 유연성: 다양한 네트워크 환경과 프로토콜에 적응할 수 있는 유연성을 제공한다.
  6. 오류 검출 및 수정: 각 계층에서 데이터의 무결성을 확인하고 필요한 경우 오류를 수정할 수 있다.

Encapsulation (캡슐화)#

데이터가 송신자로부터 수신자에게 전송될 때 발생하는 과정

특징#

각 네트워크 계층에서 데이터에 헤더(Header)나 트레일러를 추가하는 과정
OSI 7계층이나 TCP/IP 모델의 상위 계층에서 하위 계층으로 데이터가 이동할 때 발생한다.
각 계층은 자신의 프로토콜 정보를 데이터에 추가한다.

Decapsulation (역캡슐화)#

수신자 측에서 발생하는 캡슐화의 역과정

특징#

수신된 데이터로부터 각 계층의 헤더와 트레일러를 제거하는 과정
물리 계층에서 시작하여 애플리케이션 계층으로 올라가며 진행된다.
각 계층은 자신의 프로토콜 정보를 해석하고 제거한다.

Encapsulation & Decapsulation in TCP/IP Model#

Encapsulation#

  1. 응용 계층: 사용자 데이터 생성 (OSI의 응용, 표현, 세션 계층 통합)
  2. 전송 계층: TCP 세그먼트 또는 UDP 데이터그램 생성
  3. 인터넷 계층: IP 패킷 생성
  4. 네트워크 접근 계층: 프레임 생성 및 비트 스트림으로 변환 (OSI의 데이터 링크와 물리 계층 통합)

Decapsulation#

위 과정의 역순으로 진행

Encapsulation & Decapsulation in OSI 7Layers Model#

Encapsulation#

  1. 응용 계층: 사용자 데이터 생성
  2. 표현 계층: 데이터 형식 변환, 암호화 등
  3. 세션 계층: 세션 정보 추가
  4. 전송 계층: 세그먼트 또는 데이터그램 생성
  5. 네트워크 계층: 패킷 생성
  6. 데이터 링크 계층: 프레임 생성
  7. 물리 계층: 비트 스트림으로 변환

Decapsulation#

위 과정의 역순으로 진행되며, 각 계층에서 해당 헤더를 제거하고 데이터를 상위 계층으로 전달

OSI 7Layers와 TCP/IP 4계층에서의 Encapsulation & Decapsulation 차이점#

  1. 계층 수: OSI 모델은 7개 계층, TCP/IP 모델은 4개 계층으로 구성되어 있어 TCP/IP 모델에서는 일부 계층이 통합되어 처리된다.
  2. 세부 과정: OSI 모델에서는 표현 계층과 세션 계층에서 추가적인 처리가 이루어지지만, TCP/IP 모델에서는 이 과정들이 응용 계층에 통합되어 있다.
  3. 유연성: TCP/IP 모델은 더 단순화되어 있어 실제 네트워크 구현에 더 적합하며, 유연성이 높다.
  4. 프로토콜 특화: TCP/IP 모델은 특정 프로토콜(TCP, IP)에 맞춰져 있어, 이들 프로토콜의 특성을 더 잘 반영한다.
  5. 물리 및 데이터 링크 계층: OSI 모델에서는 이 두 계층이 분리되어 있지만, TCP/IP 모델에서는 네트워크 접근 계층으로 통합되어 있다.

PDU(Protocol Data Unit)와 SDU(Service Data Unit)#

PDU(Protocol Data Unit)#

정의#

동일한 통신 계층(Peer-to-Peer) 간에 교환되는 전체 데이터량을 의미한다.
실제 데이터와 그 데이터를 운반하기 위한 제어 정보가 포함된다.

구성#
1

PDU = PCI(Header / Trailer) + SDU(Payload)
  1. PCI (Protocol Control Information): 헤더 정보를 포함합니다. 발신지 주소, 수신지 주소, 순서 번호, 오류 검출용 FCS 등이 여기에 해당한다.
  2. SDU (Service Data Unit): 실제 사용자 정보를 담고 있다
계층별 PDU#

각 네트워크 계층마다 PDU의 명칭이 다르다:

물리 계층 (1계층): 비트(Bits)#
  • 단순한 0과 1의 디지털 신호로 구성된다.
  • 추가적인 구조는 없으며, 순수한 데이터 스트림이다.
데이터 링크 계층 (2계층): 프레임(Frame)#
  • 헤더: 출발지/목적지 MAC 주소, 프레임 타입 등
  • 페이로드: 상위 계층에서 받은 데이터
  • 트레일러: 오류 검출을 위한 FCS(Frame Check Sequence)
네트워크 계층 (3계층): 패킷(Packet)#
  • 헤더: 출발지/목적지 IP 주소, 프로토콜 타입, TTL(Time To Live) 등
  • 페이로드: 상위 계층에서 받은 데이터
전송 계층 (4계층): 세그먼트(Segment) - TCP / 데이터그램(Datagram) - UDP#

TCP 세그먼트 구성:

  • 헤더: 출발지/목적지 포트 번호, 시퀀스 번호, 확인 응답 번호, 윈도우 크기 등
  • 페이로드: 상위 계층에서 받은 데이터

UDP 데이터그램 구성:

  • 헤더: 출발지/목적지 포트 번호, 길이, 체크섬
  • 페이로드: 상위 계층에서 받은 데이터
세션, 표현, 응용 계층 (5-7계층): 데이터(Data) 또는 메시지(Message)#
  • 세션 계층: 세션 관리 정보가 포함된 데이터
  • 표현 계층: 인코딩, 암호화, 압축 등이 적용된 데이터
  • 응용 계층: 애플리케이션 특정 헤더와 실제 사용자 데이터
역할#
  1. 캡슐화: 상위 계층에서 하위 계층으로 데이터가 전달될 때, 각 계층은 자신의 헤더를 추가하여 PDU를 생성한다.
  2. 역캡슐화: 수신 측에서는 각 계층에서 해당 계층의 헤더를 제거하고 상위 계층으로 데이터를 전달한다.
  3. 계층 간 통신: PDU는 동일한 계층 간에 데이터를 주고받는 단위로 사용된다.
중요성#
  1. 모듈성: 각 계층의 기능을 논리적으로 분리하여 네트워크 프로토콜의 모듈식 설계를 가능하게 한다.
  2. 표준화: PDU를 통해 각 계층의 기능과 역할을 명확히 정의할 수 있다.
  3. 오류 제어: 각 계층에서 PDU를 통해 데이터의 무결성을 확인하고 필요한 경우 오류를 수정할 수 있다.

SDU(Service Data Unit)#

정의#

상위 계층에서 하위 계층으로 전달되는 데이터 단위입니다. 이는 아직 하위 계층에 의해 캡슐화되지 않은 순수한 데이터를 의미한다.

특징#
  1. 계층 간 전달: SDU는 프로토콜 스택의 한 계층에서 다른 계층으로 전달되는 데이터이다.
  2. 원본 데이터: SDU는 상위 계층에서 생성된 원본 데이터나 메시지를 포함한다.
  3. 캡슐화 이전 상태: SDU는 하위 계층에서 PDU(Protocol Data Unit)로 변환되기 전의 상태이다.
  4. 가공되지 않은 데이터: 수신 계층에서는 SDU를 가공되지 않은 원시 세그먼트로 읽을 수 있다.
크기 제한#
  • Maximum SDU Size (MSS): 각 계층에서 처리할 수 있는 최대 SDU 크기
  • SDU가 MSS를 초과할 경우 분할(segmentation) 발생
  • 네트워크 성능과 효율성에 직접적인 영향을 미침
처리 과정#
  1. 분할과 연결: SDU는 필요에 따라 여러 개의 작은 단위로다.

  • AH: Application Header
  • PH: Presentation Header
  • SH: Session Header
  • TH: Transport Header
  • NH: Network Header / NT: Network Tail
  • DH: Data Link Header / DT: Data Link Tail

구체적인 예시#

응용 계층 (Application Layer)#
  • SDU: 사용자 데이터 (예: 이메일 내용, 웹 페이지 HTML)
  • PDU: 메시지 (Message)
  • 예시: HTTP 요청에서 SDU는 웹 페이지 내용이고, PDU는 HTTP 헤더와 본문을 포함한 전체 HTTP 메시지.
표현 계층 (Presentation Layer)#
  • SDU: 응용 계층에서 받은 데이터
  • PDU: 포맷된 데이터
  • 예시: JPEG 이미지 파일에서 SDU는 원본 이미지 데이터이고, PDU는 압축된 JPEG 형식의 데이터.
세션 계층 (Session Layer)#
  • SDU: 표현 계층에서 받은 데이터
  • PDU: 세션 데이터
  • 예시: RPC(Remote Procedure Call)에서 SDU는 원격 프로시저 호출 데이터이고, PDU는 세션 관리 정보가 추가된 전체 RPC 메시지.
전송 계층 (Transport Layer)#
  • SDU: 세션 계층에서 받은 데이터
  • PDU: 세그먼트(TCP) 또는 데이터그램(UDP)
  • 예시: TCP에서 SDU는 상위 계층에서 받은 데이터이고, PDU는 TCP 헤더(출발지/목적지 포트, 시퀀스 번호 등)가 추가된 TCP 세그먼트.
네트워크 계층 (Network Layer)#
  • SDU: 전송 계층에서 받은 세그먼트 또는 데이터그램
  • PDU: 패킷
  • 예시: IP에서 SDU는 TCP 세그먼트이고, PDU는 IP 헤더(출발지/목적지 IP 주소, TTL 등)가 추가된 IP 패킷.
데이터 링크 계층 (Data Link Layer)#
  • SDU: 네트워크 계층에서 받은 패킷
  • PDU: 프레임
  • 예시: 이더넷에서 SDU는 IP 패킷이고, PDU는 이더넷 헤더(MAC 주소 등)와 트레일러(FCS)가 추가된 이더넷 프레임.
물리 계층 (Physical Layer)#
  • SDU: 데이터 링크 계층에서 받은 프레임
  • PDU: 비트(Bit)
  • 예시: SDU는 이더넷 프레임이고, PDU는 전기 신호나 광 신호로 변환된 비트 스트림.
