쓰로틀링 (Throttling)#

쓰로틀링(Throttling)은 시스템이나 리소스의 과도한 사용을 제어하고 관리하는 기술이다.
특정 시간 동안 처리할 수 있는 요청의 수나 데이터의 양을 제한함으로써 시스템의 과부하를 방지하고 안정적인 서비스를 제공하는 것이 주요 목적이다.

작동 방식#

  • 지정된 시간 간격 내에 허용된 요청 수를 초과하면 추가 요청을 거부하거나 지연시킨다.
  • 예를 들어, 초당 1000개의 요청만 허용한다면 그 이상의 요청은 throttle된다.

public class ThrottlingController {
    private final int MAX_REQUESTS_PER_SECOND = 1000;
    private final Queue<Long> requestTimestamps = new LinkedList<>();
    
    public boolean isRequestAllowed() {
        long currentTime = System.currentTimeMillis();
        
        // 1초가 지난 타임스탬프 제거
        while (!requestTimestamps.isEmpty() && 
               requestTimestamps.peek() < currentTime - 1000) {
            requestTimestamps.poll();
        }
        
        // 현재 요청 수가 제한보다 적으면 허용
        if (requestTimestamps.size() < MAX_REQUESTS_PER_SECOND) {
            requestTimestamps.offer(currentTime);
            return true;
        }
        
        return false;
    }
}

주요 용도#

  • API 사용량 제한
  • 서버 리소스 보호
  • DDoS 공격 방지
  • 네트워크 트래픽 관리

    class RateLimiter:
    def __init__(self, max_requests, time_window):
        self.max_requests = max_requests
        self.time_window = time_window
        self.requests = []

    def is_allowed(self):
        current_time = time.time()
        # 시간 창 밖의 요청 제거
        self.requests = [req for req in self.requests 
                        if req > current_time - self.time_window]

        if len(self.requests) < self.max_requests:
            self.requests.append(current_time)
            return True
        return False

    public class ConnectionThrottler {
    private final Semaphore connectionSemaphore;

    public ConnectionThrottler(int maxConnections) {
        this.connectionSemaphore = new Semaphore(maxConnections);
    }

    public boolean acquireConnection() {
        return connectionSemaphore.tryAcquire();
    }

    public void releaseConnection() {
        connectionSemaphore.release();
    }
}

    class BandwidthThrottler:
    def __init__(self, max_bandwidth):
        self.max_bandwidth = max_bandwidth
        self.current_usage = 0

    def can_transfer(self, bytes_size):
        if self.current_usage + bytes_size <= self.max_bandwidth:
            self.current_usage += bytes_size
            return True
        return False

구현 방식#

  1. Hard Throttling: 설정된 제한을 엄격하게 적용
  2. Soft Throttling: 일정 비율까지 초과 허용
  3. Elastic/Dynamic Throttling: 서버 자원 상태에 따라 유동적으로 적용

장점#

  • 시스템 안정성 향상
  • 리소스의 공정한 분배
  • 서비스 품질 유지

단점#

  • 일부 요청의 지연 또는 거부로 인한 사용자 경험 저하 가능성
  • 구현 및 설정의 복잡성

쓰로틀링은 네트워크 트래픽 관리, 클라우드 서비스, 웹 애플리케이션 등 다양한 분야에서 중요한 역할을 한다.
시스템의 안정성을 유지하면서 효율적인 리소스 관리를 가능하게 하는 기술이다.

    def calculate_threshold(self):
    # 시스템 리소스 모니터링
    cpu_usage = self.get_cpu_usage()
    memory_usage = self.get_memory_usage()
    network_load = self.get_network_load()

    # 동적 임계값 계산
    threshold = self.base_threshold
    if cpu_usage > 80:
        threshold *= 0.8
    if memory_usage > 90:
        threshold *= 0.7
    if network_load > 70:
        threshold *= 0.9

    return threshold

    public class ThrottlingExceptionHandler {
    public Response handleThrottlingException(ThrottlingException ex) {
        // 재시도 시간 계산
        long retryAfter = calculateRetryTime();

        return Response.status(429)  // Too Many Requests
                      .header("Retry-After", retryAfter)
                      .entity("요청이 제한되었습니다. " + retryAfter + 
                             "초 후에 다시 시도해주세요.")
                      .build();
    }
}

    class DatabaseConnectionThrottler:
    def __init__(self, pool_size):
        self.connection_pool = queue.Queue(maxsize=pool_size)

    def get_connection(self):
        try:
            return self.connection_pool.get(timeout=5)
        except queue.Empty:
            raise ThrottlingException("데이터베이스 연결 제한에 도달했습니다.")
