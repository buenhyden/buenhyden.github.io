폴링(Polling)#

폴링(Polling)은 소프트웨어 개발에서 중요한 통신 기법으로, 클라이언트가 주기적으로 서버에 데이터나 상태 변경을 요청하는 방식이다.
이 방법은 실시간 또는 준실시간 업데이트가 필요한 애플리케이션에서 자주 사용된다.

폴링의 작동 원리#

  1. 클라이언트가 서버에 주기적으로 요청을 보낸다.
  2. 서버는 요청을 받아 처리하고 응답을 반환한다.
  3. 클라이언트는 응답을 받아 필요한 작업을 수행한다.
  4. 일정 시간 후 클라이언트는 다시 요청을 보낸다.

이 과정은 지속적으로 반복된다.

폴링의 장점#

  1. 구현의 용이성: 폴링은 상대적으로 간단하게 구현할 수 있다.
  2. 호환성: 대부분의 환경과 기술에서 사용 가능하다.
  3. 요청 빈도 제어: 개발자가 요청 주기를 조절할 수 있어 유연성이 높다.
  4. 비실시간 업데이트에 적합: 즉각적인 업데이트가 필요하지 않은 경우에 효과적이다.

폴링의 단점#

  1. 서버 부하: 불필요한 요청으로 인해 서버에 부담을 줄 수 있다.
  2. 실시간성 부족: 폴링 주기에 따라 실시간 업데이트가 지연될 수 있다.
  3. 리소스 낭비: 변경사항이 없어도 지속적으로 요청을 보내므로 리소스가 낭비될 수 있다.

폴링의 사용 사례#

  1. 이메일 클라이언트: 새 메일 확인을 위해 주기적으로 서버에 요청을 보낸다.
  2. 주식 시세 모니터링: 일정 간격으로 최신 주가 정보를 요청한다.
  3. 소셜 미디어 피드: 새로운 게시물이나 알림을 확인하기 위해 사용된다.
  4. IoT 디바이스 모니터링: 센서 데이터를 주기적으로 확인한다.

폴링의 구현#

폴링은 주로 JavaScript의 setInterval() 함수나 setTimeout() 함수를 사용하여 구현한다.

function poll() {
  fetch('/api/data')
    .then(response => response.json())
    .then(data => {
      // 데이터 처리
      setTimeout(poll, 5000); // 5초 후 다시 요청
    });
}

poll(); // 폴링 시작

폴링의 대안#

  1. 롱 폴링(Long Polling): 서버가 즉시 응답하지 않고 이벤트 발생 시까지 연결을 유지하는 방식이다.
  2. 웹소켓(WebSocket): 양방향 실시간 통신을 지원하는 프로토콜이다.
  3. 서버-센트 이벤트(SSE): 서버에서 클라이언트로 단방향 실시간 이벤트를 전송하는 기술이다.
