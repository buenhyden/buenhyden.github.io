gRPC API#

 gRPC(gRPC Remote Procedure Call)는 Google에서 개발한 오픈소스 원격 프로시저 호출(RPC) 시스템.
 이 시스템은 효율적이고 빠른 서비스 간 통신을 제공하며, 다양한 프로그래밍 언어와 플랫폼에서 사용할 수 있다.

gRPC는 특히 다음과 같은 상황에서 유용하다:

  1. 마이크로서비스 아키텍처
    • 서비스 간 효율적인 통신
    • 강력한 타입 안정성
    • 뛰어난 성능
  2. 실시간 통신이 필요한 시스템
    • 채팅 애플리케이션
    • 실시간 모니터링
    • 게임 서버
  3. 모바일 애플리케이션
    • 효율적인 데이터 전송
    • 배터리 사용량 최적화
    • 네트워크 대역폭 절약

gRPC의 주요 특징#

  1. Protocol Buffers 사용: gRPC는 데이터 직렬화를 위해 Protocol Buffers를 사용합니다. 이는 JSON이나 XML보다 더 작고 빠른 데이터 포맷을 제공합니다.
  2. HTTP/2 기반: HTTP/2 프로토콜을 사용하여 높은 성능과 낮은 지연 시간을 제공합니다.
  3. 양방향 스트리밍: 클라이언트와 서버 간의 양방향 스트리밍을 지원하여 실시간 데이터 교환이 가능합니다.
  4. 다양한 언어 지원: Java, Python, Go, C++, Ruby 등 다양한 프로그래밍 언어를 지원합니다.
  5. 강력한 타입 시스템: Protocol Buffers를 통해 강력한 타입 시스템을 제공하여 타입 안정성을 보장합니다.

gRPC 작동 방식#

  1. 서비스 정의:.proto 파일에 서비스와 메시지 구조를 정의합니다.
  2. 코드 생성: Protocol Buffer 컴파일러를 사용하여 서버와 클라이언트 코드를 자동으로 생성합니다.
  3. 서버 구현: 생성된 코드를 기반으로 서버 비즈니스 로직을 구현합니다.
  4. 클라이언트 구현: 생성된 클라이언트 코드를 사용하여 서버와 통신합니다.

gRPC의 통신 유형#

  1. Unary RPC: 클라이언트가 단일 요청을 보내고 서버가 단일 응답을 반환합니다.

    1

    rpc GetUser (UserRequest) returns (User) {}

  2. Server Streaming RPC: 클라이언트가 요청을 보내고 서버가 스트림으로 응답합니다.

    1

    rpc StreamUserActivity (UserRequest) returns (stream UserActivity) {}

  3. Client Streaming RPC: 클라이언트가 스트림으로 요청을 보내고 서버가 단일 응답을 반환합니다.

    1

    rpc BulkUpdateUsers (stream UpdateUserRequest) returns (BulkUpdateResponse) {}

  4. Bidirectional Streaming RPC: 클라이언트와 서버가 양방향으로 스트림을 주고받습니다.

    1

    rpc ChatSession (stream ChatMessage) returns (stream ChatMessage) {}

gRPC의 장점#

  1. 높은 성능: Protocol Buffers와 HTTP/2를 사용하여 빠른 데이터 전송을 제공합니다.
  2. 언어 중립성: 다양한 프로그래밍 언어를 지원하여 다중 언어 환경에서 사용이 용이합니다.
  3. 강력한 타입 안정성: Protocol Buffers를 통해 데이터 구조를 명확히 정의할 수 있습니다.
  4. 효율적인 네트워크 사용: 이진 직렬화를 통해 네트워크 대역폭을 효율적으로 사용합니다.
  5. 실시간 통신: 양방향 스트리밍을 통해 실시간 데이터 교환이 가능합니다[5].

도입할 때 고려해야 할 사항#

  1. 브라우저 지원 제한

    • 웹 클라이언트의 경우 gRPC-Web 사용 필요
    • 프록시 서버 설정 필요

  2. 학습 곡선

    • Protocol Buffers 학습 필요
    • 새로운 도구와 패턴 적응

  3. 디버깅 복잡성

    • 바이너리 형식으로 인한 가독성 제한
    • 특수한 디버깅 도구 필요
