NodeJS#

Chrome V8 JavaScript 엔진으로 빌드된 JavaScript 런타임 환경
서버 사이드에서 JavaScript를 실행할 수 있게 해주는 플랫폼으로, 웹 서버 구축부터 네트워크 프로그래밍, 일반적인 서버 사이드 프로그래밍에 이르기까지 다양한 용도로 사용된다.

런타임 환경(Runtime Environment)
프로그램이 실행되는 동안 프로그램을 위한 ‘무대’를 제공하는 시스템.
이를 연극에 비유해보면, 런타임 환경은 배우(프로그램)가 공연할 수 있는 무대, 조명, 음향 시스템 등 모든 필요한 설비를 제공하는 극장과 같다.

  • 주요 구성 요소

    • 메모리 관리 런타임 환경은 프로그램이 사용할 메모리를 할당하고 관리.
    • 기본 라이브러리 제공 런타임 환경은 프로그램이 필요로 하는 기본적인 기능들을 제공.
    • 에러 처리 프로그램 실행 중 발생하는 오류를 감지하고 관리.
    • 하드웨어 및 운영체제와의 상호작용 프로그램과 컴퓨터 시스템 사이의 중개자 역할.

  • 중요성

  1. 추상화 제공 - 개발자가 저수준의 시스템 세부사항을 신경 쓰지 않아도 되게 함 - 플랫폼 독립적인 개발 가능
  2. 성능 최적화 - 메모리 관리 최적화 - 코드 실행 속도 향상
  3. 보안 - 프로그램이 시스템에 직접 접근하는 것을 제한 - 안전한 실행 환경 제공
  • 고려사항
    • 성능 요구사항
      • 메모리 사용량
      • 실행 속도
      • 동시성 처리 능력
    • 개발 생산성
      • 제공되는 도구와 라이브러리
      • 디버깅 용이성
    • 플랫폼 지원
      • 크로스 플랫폼 지원 여부
      • 특정 운영체제에 대한 종속성

주요 특징#

  • 이벤트 기반 아키텍처:
    Node.js는 이벤트 드리븐(Event-driven) 방식으로 동작합니다. 이는 레스토랑의 웨이터가 여러 테이블의 주문을 동시에 처리하는 것과 비슷합니다.

  • 비동기 I/O:
    입출력 작업을 비동기적으로 처리하여 시스템 자원을 효율적으로 사용합니다. 이는 한 명의 웨이터가 여러 테이블을 효율적으로 서빙하는 것과 같습니다.

  • 단일 스레드 기반:
    하나의 스레드로 여러 작업을 처리합니다. 이는 혼자서 여러 가지 일을 빠르게 전환하면서 처리하는 숙련된 요리사와 같습니다.

주요 구성 요소#

  1. V8 엔진: Google에서 개발한 고성능 JavaScript 엔진
    - Just-In-Time (JIT) 컴파일을 통해 JavaScript 코드를 빠르게 실행
    - 최신 ECMAScript 표준 지원
    - Node.js를 빠르고 효율적으로 만드는 핵심 요소
  2. libuv: 비동기 I/O를 지원하는 크로스 플랫폼 라이브러리
    - 이벤트 루프 관리
    - 비동기 I/O 작업 처리 (파일 시스템, 네트워크, 타이머 등)
    - Node.js의 비동기 특성을 지원하는 핵심 구성 요소
  3. Node.js 표준 라이브러리: 다양한 내장 모듈을 포함하여 서버 측 애플리케이션 개발에 필요한 기능을 제공.
    • HTTP 모듈: HTTP 서버 및 클라이언트 생성
    • File System (fs) 모듈: 파일 시스템 접근
    • Stream 모듈: 데이터의 효율적인 읽기/쓰기 처리
    • Events 모듈: 이벤트 기반 프로그래밍 지원
  4. C++ 바인딩: C++로 작성된 네이티브 코드와 JavaScript 코드 간의 상호 작용을 가능하게 하는 바인딩을 제공. 이를 통해 성능이 중요한 작업을 네이티브 코드로 구현할 수 있다.
  5. JavaScript Core API: Node.js API를 구현하는 코어 부분으로, Node.js의 기능을 JavaScript로 사용할 수 있게 해준다.
  6. http-parser: HTTP 메시지 파싱
  7. c-ares: DNS 쿼리 처리
  8. OpenSSL: 암호화 기능
  9. zlib: 압축과 해제 기능

구조와 동작 방식#

  1. 이벤트 루프가 시작되면 비동기 작업들이 백그라운드에서 실행됩니다.
  2. 비동기 작업이 완료되면 콜백 함수가 이벤트 큐에 추가됩니다.
  3. 이벤트 루프는 큐에서 콜백을 꺼내 실행합니다.

  1. 콜 스택 (Call Stack)
    콜 스택은 프로그램이 실행하는 코드의 순서를 추적합니다. 마치 웨이터가 들고 있는 주문서와 같습니다.

function prepareDrink() {
    console.log('음료를 준비합니다');
}

orderDrink();
// 콜 스택: orderDrink -> prepareDrink -> empty

  2. 이벤트 큐 (Event Queue)
    완료된 비동기 작업들의 콜백이 대기하는 곳입니다. 음식이 준비되어 서빙을 기다리는 주방 카운터와 같습니다.

setTimeout(() => console.log('테이블1 음식 준비완료'), 1000);
setTimeout(() => console.log('테이블2 음식 준비완료'), 500);
console.log('주문을 주방으로 전달했습니다');

  3. 마이크로태스크 큐 (Microtask Queue)
    프로미스와 같은 높은 우선순위 작업을 위한 특별한 대기열입니다. VIP 고객의 주문과 같이 우선적으로 처리됩니다.

이벤트 루프의 실행 순서와 우선순위:

  1. 동기 코드가 먼저 실행됩니다 (콜 스택)
  2. 마이크로태스크 큐의 작업이 처리됩니다 (프로미스 등)
  3. 매크로태스크 큐의 작업이 처리됩니다 (타이머, I/O 등)

장점#

  1. 높은 성능
    • 비동기 처리로 인한 빠른 처리 속도
    • 효율적인 리소스 사용
  2. 생산성
    • 같은 언어로 프론트엔드와 백엔드 개발
    • 풍부한 패키지 생태계
  3. 확장성
    • 마이크로서비스 아키텍처에 적합
    • 실시간 애플리케이션 개발 용이

단점#

  1. CPU 집약적 작업에 부적합
    • 단일 스레드의 한계
    • 복잡한 연산에 취약
  2. 콜백 지옥 가능성
    • 비동기 코드의 복잡성
    • Promise와 async/await로 개선 가능
  3. 불안정한 패키지 품질
    • 써드파티 패키지의 품질 편차
    • 보안 취약점 가능성
