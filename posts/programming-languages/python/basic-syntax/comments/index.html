Comments#

주석은 코드에 대한 설명이나 메모를 추가하는 데 사용되는 텍스트이다.
주석은 코드의 가독성을 높이고, 코드의 목적과 작동 방식을 설명하며, 향후 개발자들을 위한 정보를 제공한다.

주요 목적:

  • 코드 가독성 향상
  • 코드 설명 및 문서화
  • 디버깅 시 활용
  • 코드 협업 시 이해도 증진

주석 작성 방법#

  1. 한 줄 주석
    한 줄 주석은 ‘#’ 기호로 시작하며, 해당 줄의 끝까지 주석으로 처리된다.
    예시:

    1
2

    # 이것은 한 줄 주석입니다 
print("Hello, World!") # 이것도 한 줄 주석입니다`

  2. 여러 줄 주석
    Python에는 공식적인 여러 줄 주석 문법이 없지만, 다음과 같은 방법을 사용할 수 있다.

    • 여러 줄에 ‘#‘을 사용:

      1
2

      # 이것은 여러 줄
# 주석의 예시입니다

    • 삼중 따옴표 사용:

      1
2
3
4

      """
이것은 여러 줄 주석으로
사용될 수 있는 문자열입니다
"""

주석 작성을 위한 주요 원칙#

주석을 작성할 때의 추가적인 팁:

  • 주석은 최신 상태를 유지해야 합니다. 코드가 변경되면 관련 주석도 업데이트해야 합니다.
  • 너무 자명한 내용에 대한 주석은 피합니다. 코드 자체가 자명한 경우 주석은 불필요할 수 있습니다.
  • 일관된 형식을 사용합니다. 팀이나 프로젝트의 주석 작성 규칙을 따릅니다.
  • 영어로 작성하는 것이 일반적이지만, 팀의 상황에 따라 한글로 작성하는 것도 가능합니다.

이러한 주석 작성 방법과 원칙들을 잘 활용하면 코드의 품질을 크게 향상시킬 수 있습니다. 특히 팀 프로젝트에서는 다른 개발자들이 코드를 이해하고 유지보수하는 데 큰 도움이 됩니다.

주석 작성 모범 사례#

  1. 명확하고 간결하게 작성한다.

  2. 불필요하거나 명백한 내용은 피한다.

  3. 코드의 ‘왜’에 초점을 맞춘다. ‘무엇’이 아니다.

  4. 일관된 형식과 스타일을 사용한다.

  5. 복잡한 알고리즘이나 로직에 대해 설명한다.

  6. 주석을 최신 상태로 유지한다.

  7. 코드의 목적과 의도를 설명한다.
    단순히 코드가 ‘무엇을’ 하는지가 아니라 ‘왜’ 그렇게 하는지를 설명하는 것이 중요하다.

주의사항#

  1. 과도한 주석은 코드 품질을 저하시킬 수 있다.
  2. 주석은 코드 변경 시 함께 업데이트해야 한다.
  3. 주석은 코드의 대체제가 아니다. 항상 깨끗하고 이해하기 쉬운 코드를 작성하려고 노력한다.

참고 및 출처#

Docstring

Docstring python에서 함수, 클래스, 모듈에 대한 문서화를 위해 사용되는 문자열. 이는 코드의 가독성을 높이고 다른 개발자들이 코드를 이해하는 데 도움을 준다. 주요 특징 큰따옴표 세 개(""") 또는 작은따옴표 세 개(’’’)로 둘러싸인 문자열이다. 함수, 클래스, 모듈의 첫 번째 문장으로 위치한다. __doc__ 속성을 통해 프로그램 실행 중에 접근할 수 있다. 내장 함수 help()를 통해 문서를 볼 수 있다. 기능 코드의 목적과 동작을 설명한다. 함수의 매개변수, 반환값, 예외 등을 문서화한다. 모듈이나 클래스의 전반적인 기능을 설명한다. 자동 문서 생성 도구(예: Sphinx)를 통해 API 문서를 생성할 수 있다. 고려해야 할 중요한 점들 일관성: 프로젝트 전체에서 동일한 스타일을 사용해야 한다. 명확성: 설명은 간단하고 명확해야 하며, 예시가 있으면 더 좋다. 완전성: 모든 매개변수, 반환값, 예외 상황을 문서화해야 한다. 최신성: 코드가 변경될 때 Docstring도 함께 업데이트해야 한다. 활용 방법 1 2 3 4 5 6 # Docstring 확인하기 help(google_style) # help() 함수 사용 print(google_style.__doc__) # __doc__ 속성 직접 접근 # 대화형 셸에서 사용 >>> google_style? # IPython/Jupyter에서 자동 문서 생성을 위한 도구들 Sphinx: Python 프로젝트의 표준 문서화 도구. pdoc: 간단한 API 문서를 자동으로 생성한다. MkDocs: Markdown 기반의 문서 생성 도구이다. 각 스타일은 프로젝트의 성격이나 팀의 선호도에 따라 선택할 수 있다. 중요한 것은 프로젝트 내에서 일관성 있게 사용하는 것. 또한, IDE나 문서 생성 도구와의 호환성을 고려하여 선택하는 것이 좋다. ...
