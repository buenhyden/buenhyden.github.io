컴파일러(Compiler)#

우리가 작성한 프로그래밍 언어(고급 언어)를 컴퓨터가 이해할 수 있는 기계어(저급 언어)로 번역해주는 특별한 프로그램.

컴파일러의 기능과 역할#

  1. 코드 최적화:
    컴파일러는 우리가 작성한 코드를 더 효율적으로 실행될 수 있게 개선한다.
    예를 들어:
1

result = 5 * 60 * 24  # 하루의 초 계산

이런 코드를 컴파일러는 다음과 같이 최적화할 수 있다:

1

result = 7200  # 미리 계산된 값 사용

  1. 오류 검사:
    코드를 실행하기 전에 문제점을 미리 찾아준다.
    마치 글을 출판하기 전에 교정을 보는 것과 비슷하다.

  2. 보안:
    원본 코드를 기계어로 변환하므로 프로그램의 내부 로직을 숨길 수 있다.

특징과 장점#

  1. 실행 속도: 컴파일된 프로그램은 일반적으로 더 빠르게 실행된다.
  2. 오류 검출: 컴파일 과정에서 많은 오류를 미리 발견할 수 있다.
  3. 플랫폼 최적화: 특정 하드웨어나 운영 체제에 맞춰 최적화할 수 있다.

컴파일러의 작동 과정#

컴파일러는 다음과 같은 단계로 작동한다:

  1. 어휘 분석 (Lexical Analysis):
    프로그래머가 작성한 코드를 의미 있는 조각(토큰)으로 나눈다.
    예를 들어:
1

total = price + tax

이 코드는 ’total’, ‘=’, ‘price’, ‘+’, ’tax’라는 토큰들로 나뉩니다.

  1. 구문 분석 (Syntax Analysis):
    토큰들이 프로그래밍 언어의 규칙에 맞게 작성되었는지 확인한다.
    마치 한국어 문장에서 주어, 목적어, 서술어의 순서가 올바른지 검사하는 것과 비슷하다.

  2. 의미 분석 (Semantic Analysis):
    코드의 의미가 올바른지 검사한다.
    예를 들어, 숫자와 문자열을 더하려고 하는 것과 같은 논리적 오류를 찾아낸다.

  3. 코드 생성 (Code Generation):
    최종적으로 컴퓨터가 이해할 수 있는 기계어로 변환한다.

컴파일러의 종류#

컴파일러는 크게 처리 방식, 생성되는 코드의 형태, 그리고 최적화 수준에 따라 다양한 종류로 나눌 수 있다.

각 컴파일러는 특정한 용도와 장단점을 가지고 있다.
예를 들어, 네이티브 컴파일러는 최고의 성능을 제공하지만 플랫폼 이식성이 낮다.
반면 JIT 컴파일러는 플랫폼 독립적이지만 초기 실행 속도가 상대적으로 느리다.

이러한 다양한 컴파일러들은 현대 소프트웨어 개발에서 상호 보완적으로 사용된다.
예를 들어, 한 프로젝트에서 TypeScript 트랜스파일러로 코드를 JavaScript로 변환하고, 이를 다시 V8 엔진의 JIT 컴파일러가 실행 시점에 최적화하는 식.

처리 방식에 따른 분류#

네이티브 컴파일러(Native Compiler)#

이는 가장 전통적인 형태의 컴파일러.
소스 코드를 직접 특정 CPU 아키텍처의 기계어로 변환한다.
예를 들어, GCC(GNU Compiler Collection)나 MSVC(Microsoft Visual C++ Compiler)가 여기에 속한다.
이러한 컴파일러는 프로그램을 실행하기 전에 전체 코드를 미리 컴파일한다.

C++ 코드가 네이티브 컴파일러를 통해 처리되는 예시:

크로스 컴파일러(Cross Compiler)#

한 플랫폼에서 다른 플랫폼을 위한 코드를 생성하는 컴파일러.
예를 들어, Windows PC에서 안드로이드 앱을 개발할 때 사용하는 Android NDK가 여기에 속한다.
임베디드 시스템 개발에서 특히 중요한 역할을 한다.

JIT(Just-In-Time) 컴파일러#

프로그램 실행 중에 필요한 부분만 실시간으로 기계어로 변환하는 컴파일러.
Java의 HotSpot JVM이 대표적인 예시.
다음과 같은 코드가 있다면:

1
2
3
4
5
6
7

public class Example {
    public static void main(String[] args) {
        for (int i = 0; i < 1000000; i++) {
            heavyComputation();  // 자주 호출되는 메소드
        }
    }
}

JIT 컴파일러는 heavyComputation 메소드가 자주 호출됨을 감지하고, 이를 최적화된 기계어로 컴파일하여 성능을 향상시킨다.

AOT(Ahead-Of-Time) 컴파일러#

실행 전에 중간 코드를 기계어로 변환하는 컴파일러.
Android의 ART(Android Runtime)가 대표적인 예시.
이는 앱의 설치 시점에 바이트코드를 기계어로 변환하여 저장한다.

소스 대 소스 컴파일러(Source-to-Source Compiler 또는 트랜스파일러)#

한 프로그래밍 언어를 다른 언어로 변환하는 컴파일러.

최적화 수준에 따른 분류#

최적화 컴파일러#

코드의 성능을 향상시키기 위해 다양한 최적화 기법을 적용하는 컴파일러.

LLVM이 대표적인 예시로, 다음과 같은 최적화를 수행한다:

  1. 루프 최적화: 반복문의 효율성 향상
  2. 인라인 확장: 함수 호출 오버헤드 제거
  3. 상수 폴딩: 컴파일 시점에 계산 가능한 식을 미리 계산
  4. 데드 코드 제거: 사용되지 않는 코드 제거
디버그 컴파일러#

디버깅 정보를 포함하여 컴파일하는 컴파일러.
최적화를 최소화하고 소스 코드와의 매핑 정보를 유지한다.

현대 컴파일러의 발전 방향#

  1. 실시간 최적화:
    프로그램이 실행되는 동안 계속해서 성능을 개선한다.
    마치 운전하면서 실시간으로 최적의 경로를 찾아주는 내비게이션과 비슷하다.

  2. 크로스 플랫폼 지원:
    하나의 코드로 여러 다른 기기(PC, 스마트폰, 태블릿 등)에서 실행될 수 있게 해준다.

  3. 개발자 도움 기능:
    더 나은 코드를 작성할 수 있도록 제안을 해주거나, 보안 취약점을 미리 경고해준다.
