Batch Processing#

배치 처리는 API 설계에서 중요한 통합 패턴으로, 시스템 효율성과 성능을 크게 향상시킬 수 있는 방법이다.

배치 처리의 개념#

배치 처리는 여러 개별 API 요청을 하나의 그룹으로 묶어 단일 요청으로 처리하는 방식이다. 이는 여러 개의 개별 API 호출을 하는 대신, 다수의 작업을 포함한 하나의 요청을 보내는 것이다.

예를 들어, 사용자 정보를 100개 업데이트해야 할 때:

  • 개별 처리: 100번의 API 호출 필요
  • 배치 처리: 100개의 업데이트를 포함한 1번의 API 호출만 필요

배치 처리의 주요 이점#

  1. 네트워크 오버헤드 감소: 연결 설정 및 종료 횟수가 줄어들어 네트워크 트래픽이 감소한다.
  2. 처리량 증가: 동일한 시간 내에 더 많은 작업을 처리할 수 있다.
  3. 리소스 효율성: 서버와 클라이언트 모두에서 리소스 사용이 최적화된다.
  4. 트랜잭션 일관성: 여러 작업을 하나의 트랜잭션으로 처리할 수 있어 데이터 일관성이 향상된다.
  5. 오류 처리 개선: 배치 내 작업별 상태를 추적하여 부분 실패를 효과적으로 관리할 수 있다.

// JavaScript에서 트랜잭션 관리 예시
async function processBatchWithTransaction(operations) {
  const transaction = await db.beginTransaction();
  
  try {
    const results = [];
    
    for (const op of operations) {
      try {
        const result = await executeOperation(op, transaction);
        results.push(result);
      } catch (error) {
        // All-or-Nothing 방식에서는 여기서 예외를 다시 던짐
        throw error;
        
        // Best-Effort 방식에서는 오류를 기록하고 계속 진행
        // results.push({ status: 500, error: error.message });
      }
    }
    
    await transaction.commit();
    return results;
  } catch (error) {
    await transaction.rollback();
    throw error;
  }
}

@app.route('/api/batch', methods=['POST'])
def handle_batch():
    operations = request.json.get('operations', [])
    results = []
    
    for op in operations:
        method = op.get('method')
        path = op.get('path')
        body = op.get('body', {})
        
        # 내부적으로 각 작업 처리
        result = process_operation(method, path, body)
        results.append(result)
    
    return jsonify({"results": results})

// Apollo Server에서 배치 요청 처리
const server = new ApolloServer({
  schema,
  plugins: [
    {
      requestDidStart() {
        return {
          executionDidStart({ document }) {
            // document에는 여러 쿼리/뮤테이션이 포함될 수 있음
            // 데이터 로더를 통해 N+1 문제 해결
            const userLoader = new DataLoader(keys => fetchUsersByIds(keys));
            
            return {
              willResolveField({ source, args, context }) {
                if (source && source.userId) {
                  // 배치 처리된 사용자 로딩
                  return userLoader.load(source.userId);
                }
              }
            };
          }
        };
      }
    }
  ]
});

배치 처리의 고려사항#

  1. 성능과 최적화

    • 데이터베이스 최적화: 배치 작업을 위한 벌크 연산 활용 (INSERT, UPDATE 등)
    • 메모리 관리: 대규모 배치의 경우 스트리밍 처리 고려
    • 병렬 처리: 독립적인 작업은 병렬로 처리하여 성능 향상

  2. 보안 고려사항

    • 인증과 권한: 모든 배치 작업에 대한 권한 검증 필요
    • 입력 검증: 배치 내 모든 작업 데이터 철저히 검증
    • 속도 제한: 배치 API에 대한 속도 제한 정책 구현

  3. 오류 처리 전략

    • 부분 성공 처리: 일부 작업 실패 시 대응 방식 결정
    • 중복 요청 처리: 멱등성 보장으로 안전한 재시도 지원
    • 롤백 메커니즘: 트랜잭션 롤백 및 복구 전략 수립
