Memento Pattern#

Mediator 패턴은 객체 지향 소프트웨어 디자인 패턴 중 하나로, 객체들 간의 복잡한 상호작용을 캡슐화하여 객체 간 결합도를 낮추는 행위 패턴이다.

Mediator 패턴은 객체들 간의 직접적인 통신을 제한하고, 중재자 객체를 통해 간접적으로 상호작용하도록 하는 패턴이다. 주요 목적은 다음과 같다:

  1. 객체 간 결합도 감소
  2. 객체 간 상호작용의 중앙 집중화
  3. 코드의 재사용성과 유지보수성 향상

Mediator 패턴은 객체 간 상호작용이 복잡해질 때 특히 유용하다. 이 패턴을 통해 시스템의 유지보수성을 향상시키고, 객체 간 결합도를 낮출 수 있다. 하지만 Mediator 객체가 너무 복잡해지지 않도록 주의해야 한다.

Mediator 패턴의 구조#

  1. Mediator: 객체들 간의 상호작용을 정의하는 인터페이스
  2. ConcreteMediator: Mediator 인터페이스를 구현하는 클래스
  3. Colleague: Mediator와 통신하는 객체들의 인터페이스
  4. ConcreteColleague: Colleague 인터페이스를 구현하는 클래스들

구현 방법#

  1. Mediator 인터페이스 정의
  2. ConcreteMediator 클래스 구현
  3. Colleague 인터페이스 정의
  4. ConcreteColleague 클래스들 구현

// Mediator 인터페이스
interface ChatMediator {
    void sendMessage(String message, User user);
    void addUser(User user);
}

// ConcreteMediator
class ChatRoom implements ChatMediator {
    private List<User> users;

    public ChatRoom() {
        this.users = new ArrayList<>();
    }

    @Override
    public void sendMessage(String message, User user) {
        for (User u : users) {
            if (u != user) {
                u.receive(message);
            }
        }
    }

    @Override
    public void addUser(User user) {
        this.users.add(user);
    }
}

// Colleague 인터페이스
abstract class User {
    protected ChatMediator mediator;
    protected String name;

    public User(ChatMediator mediator, String name) {
        this.mediator = mediator;
        this.name = name;
    }

    public abstract void send(String message);
    public abstract void receive(String message);
}

// ConcreteColleague
class ChatUser extends User {
    public ChatUser(ChatMediator mediator, String name) {
        super(mediator, name);
    }

    @Override
    public void send(String message) {
        System.out.println(this.name + " : Sending Message = " + message);
        mediator.sendMessage(message, this);
    }

    @Override
    public void receive(String message) {
        System.out.println(this.name + " : Received Message : " + message);
    }
}

Mediator 패턴의 장점#

  1. 객체 간 결합도 감소: 객체들이 직접 통신하지 않아 유연성이 향상된다.
  2. 코드 재사용성 증가: 객체들의 상호작용 로직을 분리하여 재사용이 용이해진다.
  3. 단일 책임 원칙 준수: Mediator가 통신을 담당하고, 각 객체는 자신의 역할에만 집중할 수 있다.

Mediator 패턴의 단점#

  1. Mediator 객체의 복잡성 증가: 시스템이 커질수록 Mediator 객체가 복잡해질 수 있다.
  2. 성능 저하 가능성: 모든 통신이 Mediator를 거치므로 성능에 영향을 줄 수 있다.

사용 시나리오#

  1. GUI 애플리케이션: 다양한 UI 컴포넌트 간의 상호작용 관리.
  2. 항공 관제 시스템: 비행기들 간의 통신 조정.
  3. 채팅 애플리케이션: 사용자들 간의 메시지 교환 관리.
