Metaheuristic

메타휴리스틱은 여러 휴리스틱 방법들을 체계적으로 활용하거나 관리하는 상위 수준의 전략이라고 볼 수 있다.

Metaheuristic#

메타휴리스틱 알고리즘은 현실 세계의 복잡한 최적화 문제를 해결하기 위한 강력한 도구이다.
다양한 문제에 적용할 수 있는 유연성과 합리적인 시간 내에 좋은 해결책을 제공하는 능력으로 인해, 학계와 산업계 모두에서 널리 활용되고 있다.

초보 개발자라면 기본적인 메타휴리스틱 알고리즘(시뮬레이티드 어닐링, 유전 알고리즘 등)부터 시작하여, 점차 더 복잡한 알고리즘과 하이브리드 접근법으로 발전해 나가는 것이 좋다. 이러한 알고리즘의 기본 원리를 이해하고 실제 문제에 적용하는 능력은 다양한 분야에서 가치 있는 기술이 될 것이다.

메타휴리스틱(Metaheuristic)은 최적화 문제를 해결하기 위한 고수준 문제 해결 방법론이다.
‘메타(Meta)‘는 ‘상위의’ 또는 ‘더 높은 수준의’라는 의미를 가지며, ‘휴리스틱(Heuristic)‘은 ‘경험에 기반한 문제 해결 방법’을 의미한다.
따라서 메타휴리스틱은 여러 휴리스틱 방법들을 체계적으로 활용하거나 관리하는 상위 수준의 전략이라고 볼 수 있다.

쉽게 말해, 메타휴리스틱은 복잡하고 어려운 최적화 문제를 푸는 데 사용되는 일반적인 전략의 모음이다.
이 방법들은 정확한 해결책을 보장하지는 않지만, 합리적인 시간 내에 좋은 품질의 해결책을 찾는 것을 목표로 한다.

메타휴리스틱이 중요한 이유#

현실 세계의 많은 문제는 복잡하고 계산적으로 풀기 어렵다.
예를 들어, 여행하는 판매원 문제(TSP), 스케줄링 문제, 네트워크 설계 등은 가능한 해결책의 수가 너무 많아서 모든 경우를 확인하는 것이 실질적으로 불가능하다.
이러한 문제들은 종종 NP-hard 또는 NP-complete 문제로 분류된다.

메타휴리스틱 알고리즘은 이런 복잡한 문제들에 대해:

  • 합리적인 시간 내에 좋은 해결책을 제공한다.
  • 다양한 문제에 적용 가능한 유연성을 갖는다.
  • 지역 최적값(local optima)에 빠지는 것을 피하는 메커니즘을 제공한다.
  • 복잡한 탐색 공간을 효과적으로 탐색할 수 있다.

3. 메타휴리스틱의 기본 원리#

대부분의 메타휴리스틱 알고리즘은 다음과 같은 기본 원리를 따릅니다:

3.1. 탐색과 활용의 균형(Exploration vs. Exploitation)#

  • 탐색(Exploration): 아직 방문하지 않은 해결책 공간의 다양한 영역을 탐색하는 것입니다. 이는 새로운 가능성을 발견하는 데 중요합니다.
  • 활용(Exploitation): 현재까지 발견한 좋은 해결책 주변을 더 자세히 탐색하는 것입니다. 이는 현재의 좋은 해결책을 개선하는 데 중요합니다.

좋은 메타휴리스틱 알고리즘은 이 두 가지 전략 사이의 균형을 효과적으로 유지합니다.

3.2. 이웃 생성 및 이동(Neighborhood Generation and Movement)#

대부분의 메타휴리스틱은 현재 해결책의 ‘이웃’을 생성하고, 특정 기준에 따라 해당 이웃으로 이동하는 과정을 반복합니다. 이웃은 현재 해결책에 작은 변화를 가한 새로운 해결책입니다.

3.3. 메모리 사용(Memory Usage)#

메타휴리스틱은 탐색 과정에서 얻은 정보를 활용하는 방식에 따라 분류될 수 있습니다:

  • 기억이 없는(Memoryless) 방법: 현재 상태만을 고려합니다(예: 단순 언덕 오르기).
  • 단기 기억(Short-term Memory) 방법: 최근의 이동과 결과를 기억합니다(예: 타부 탐색).
  • 장기 기억(Long-term Memory) 방법: 전체 탐색 과정의 패턴을 기억합니다(예: 개미 군집 최적화).

4. 주요 메타휴리스틱 알고리즘#

이제 가장 널리 사용되는 메타휴리스틱 알고리즘들을 살펴보겠습니다:

4.1. 시뮬레이티드 어닐링(Simulated Annealing)#

시뮬레이티드 어닐링은 금속 공학에서 영감을 받은 알고리즘으로, 금속이 천천히 냉각되면서 최적의 에너지 상태를 찾아가는 과정을 모방합니다.

기본 원리#
  1. 초기 해결책과 초기 온도를 설정합니다.
  2. 현재 해결책의 이웃을 생성합니다.
  3. 이웃이 현재 해결책보다 좋으면 항상 이동합니다.
  4. 이웃이 더 나쁘더라도 확률적으로 이동할 수 있습니다(이 확률은 온도에 의존하며, 온도가 낮아질수록 감소).
  5. 온도를 낮추고 2~4단계를 반복합니다.
  6. 종료 조건(예: 충분히 낮은 온도)에 도달하면 중단합니다.

import math
import random

def simulated_annealing(initial_solution, cost_function, neighbor_function, temperature, cooling_rate, max_iterations):
    current_solution = initial_solution
    current_cost = cost_function(current_solution)
    best_solution = current_solution
    best_cost = current_cost
    
    iteration = 0
    while iteration < max_iterations and temperature > 0.1:
        # 이웃 해결책 생성
        neighbor = neighbor_function(current_solution)
        neighbor_cost = cost_function(neighbor)
        
        # 비용 차이 계산
        cost_difference = neighbor_cost - current_cost
        
        # 이웃이 더 좋거나 확률적으로 수락
        if cost_difference < 0 or random.random() < math.exp(-cost_difference / temperature):
            current_solution = neighbor
            current_cost = neighbor_cost
            
            # 최적 해결책 업데이트
            if current_cost < best_cost:
                best_solution = current_solution
                best_cost = current_cost
        
        # 온도 감소
        temperature *= cooling_rate
        iteration += 1
    
    return best_solution, best_cost

시뮬레이티드 어닐링의 장점은 지역 최적값에 빠지는 것을 피할 수 있다는 것입니다. 초기에는 나쁜 이동도 수락할 확률이 높아 다양한 영역을 탐색하고, 알고리즘이 진행됨에 따라 점점 더 좋은 해결책에 집중합니다.

4.2. 유전 알고리즘(Genetic Algorithm)#

유전 알고리즘은 자연 선택과 유전학에서 영감을 받은 방법으로, 해결책의 ‘집단’이 세대를 거듭하며 진화하는 과정을 모방합니다.

기본 원리#
  1. 초기 해결책 집단(개체군)을 무작위로 생성합니다.
  2. 각 해결책의 적합도(목적 함수의 값)를 평가합니다.
  3. 선택: 적합도가 높은 해결책들을 더 자주 선택합니다.
  4. 교차: 선택된 해결책들을 짝지어 ‘유전적 교차’를 통해 새로운 해결책을 생성합니다.
  5. 변이: 일정 확률로 새 해결책에 작은 무작위 변화를 가합니다.
  6. 새로운 세대가 이전 세대를 대체합니다.
  7. 종료 조건에 도달할 때까지 2~6단계를 반복합니다.

import random

def genetic_algorithm(population_size, chromosome_length, fitness_function, selection_function, crossover_function, mutation_function, mutation_rate, max_generations):
    # 초기 개체군 생성
    population = [generate_random_chromosome(chromosome_length) for _ in range(population_size)]
    
    for generation in range(max_generations):
        # 적합도 평가
        fitness_scores = [fitness_function(chromosome) for chromosome in population]
        
        # 최고 적합도 기록
        best_idx = fitness_scores.index(max(fitness_scores))
        best_chromosome = population[best_idx]
        
        # 새 개체군 생성
        new_population = []
        
        while len(new_population) < population_size:
            # 선택
            parent1 = selection_function(population, fitness_scores)
            parent2 = selection_function(population, fitness_scores)
            
            # 교차
            child = crossover_function(parent1, parent2)
            
            # 변이
            if random.random() < mutation_rate:
                child = mutation_function(child)
            
            new_population.append(child)
        
        # 세대 교체
        population = new_population
    
    # 최종 개체군에서 최적 해결책 찾기
    final_fitness_scores = [fitness_function(chromosome) for chromosome in population]
    best_idx = final_fitness_scores.index(max(final_fitness_scores))
    
    return population[best_idx], max(final_fitness_scores)

유전 알고리즘의 장점은 해결책 공간을 병렬적으로 탐색하여 다양한 영역을 효과적으로 탐색할 수 있다는 점입니다. 또한 복잡한 문제에 대해 자연스럽게 적응할 수 있습니다.

4.3. 타부 탐색(Tabu Search)#

타부 탐색은 이미 방문한 해결책을 ‘타부(금지)‘로 지정하여 같은 지역을 반복적으로 탐색하는 것을 방지하는 메모리 기반 전략입니다.

기본 원리#
  1. 초기 해결책에서 시작합니다.
  2. 현재 해결책의 모든 이웃을 생성합니다.
  3. 타부 목록에 없는 이웃 중에서 가장 좋은 해결책으로 이동합니다(심지어 현재보다 나쁘더라도).
  4. 방금 방문한 해결책(또는 특정 이동)을 타부 목록에 추가합니다.
  5. 필요에 따라 타부 목록을 업데이트합니다(오래된 항목 제거).
  6. 종료 조건에 도달할 때까지 2~5단계를 반복합니다.

def tabu_search(initial_solution, cost_function, neighborhood_function, tabu_size, max_iterations):
    current_solution = initial_solution
    current_cost = cost_function(current_solution)
    best_solution = current_solution
    best_cost = current_cost
    
    # 타부 목록 초기화
    tabu_list = []
    
    for iteration in range(max_iterations):
        # 모든 이웃 생성
        neighbors = neighborhood_function(current_solution)
        
        # 가장 좋은 이웃 찾기 (타부 목록에 없는)
        best_neighbor = None
        best_neighbor_cost = float('inf')
        
        for neighbor in neighbors:
            if neighbor not in tabu_list:
                neighbor_cost = cost_function(neighbor)
                if neighbor_cost < best_neighbor_cost:
                    best_neighbor = neighbor
                    best_neighbor_cost = neighbor_cost
        
        # 이웃이 없으면 종료
        if best_neighbor is None:
            break
        
        # 이동
        current_solution = best_neighbor
        current_cost = best_neighbor_cost
        
        # 최적 해결책 업데이트
        if current_cost < best_cost:
            best_solution = current_solution
            best_cost = current_cost
        
        # 타부 목록 업데이트
        tabu_list.append(current_solution)
        if len(tabu_list) > tabu_size:
            tabu_list.pop(0)  # 가장 오래된 항목 제거
    
    return best_solution, best_cost

타부 탐색의 장점은 지역 최적값에서 벗어날 수 있는 메커니즘을 제공하면서도, 탐색 과정에서 학습한 정보를 활용한다는 점입니다.

4.4. 입자 군집 최적화(Particle Swarm Optimization, PSO)#

입자 군집 최적화는 새 떼나 물고기 떼와 같은 사회적 행동에서 영감을 받은 알고리즘으로, 해결책들(‘입자들’)이 해결책 공간을 ‘날아다니며’ 서로의 경험에서 배웁니다.

기본 원리#
  1. 입자들(해결책들)의 집단을 무작위로 초기화합니다.
  2. 각 입자는 위치(현재 해결책)와 속도(이동 방향과 크기)를 가집니다.
  3. 각 입자는 자신의 최고 위치와 전체 군집의 최고 위치를 기억합니다.
  4. 각 반복에서, 입자들은 다음에 따라 속도를 조정합니다:
    • 자신의 최고 위치를 향한 이동(인지적 구성 요소)
    • 군집의 최고 위치를 향한 이동(사회적 구성 요소)
  5. 새로운 속도에 따라 입자들의 위치를 업데이트합니다.
  6. 종료 조건에 도달할 때까지 2~5단계를 반복합니다.

import numpy as np

def particle_swarm_optimization(fitness_function, dimensions, population_size, max_iterations, c1=2.0, c2=2.0, w=0.7):
    # 입자 위치와 속도 초기화
    positions = np.random.rand(population_size, dimensions)
    velocities = np.random.rand(population_size, dimensions) * 0.1
    
    # 개인 최적 위치와 적합도 초기화
    personal_best_positions = positions.copy()
    personal_best_fitnesses = np.array([fitness_function(p) for p in positions])
    
    # 전역 최적 위치와 적합도 초기화
    global_best_idx = np.argmax(personal_best_fitnesses)
    global_best_position = personal_best_positions[global_best_idx].copy()
    global_best_fitness = personal_best_fitnesses[global_best_idx]
    
    for iteration in range(max_iterations):
        # 각 입자 업데이트
        for i in range(population_size):
            # 속도 업데이트
            r1, r2 = np.random.rand(), np.random.rand()
            cognitive_velocity = c1 * r1 * (personal_best_positions[i] - positions[i])
            social_velocity = c2 * r2 * (global_best_position - positions[i])
            velocities[i] = w * velocities[i] + cognitive_velocity + social_velocity
            
            # 위치 업데이트
            positions[i] += velocities[i]
            
            # 경계 처리 (0-1 사이로 제한)
            positions[i] = np.clip(positions[i], 0, 1)
            
            # 적합도 평가
            fitness = fitness_function(positions[i])
            
            # 개인 최적 위치 업데이트
            if fitness > personal_best_fitnesses[i]:
                personal_best_fitnesses[i] = fitness
                personal_best_positions[i] = positions[i].copy()
                
                # 전역 최적 위치 업데이트
                if fitness > global_best_fitness:
                    global_best_fitness = fitness
                    global_best_position = positions[i].copy()
    
    return global_best_position, global_best_fitness

PSO의 장점은 구현이 비교적 간단하면서도 연속적인 최적화 문제에 매우 효과적이라는 점입니다. 또한 병렬 처리에 적합하며, 복잡한 다차원 공간에서도 좋은 성능을 보입니다.

4.5. 개미 군집 최적화(Ant Colony Optimization, ACO)#

개미 군집 최적화는 실제 개미들이 먹이를 찾아 최적의 경로를 발견하는 방식에서 영감을 받은 알고리즘입니다. 개미들은 페로몬이라는 화학 물질을 통해 서로 정보를 공유합니다.

기본 원리#
  1. 가상의 개미들을 그래프의 노드에 배치합니다.
  2. 각 개미는 페로몬 수준과 휴리스틱 정보를 기반으로 다음 노드를 확률적으로 선택합니다.
  3. 모든 개미가 전체 경로를 완성하면, 경로의 품질에 비례하여 페로몬을 업데이트합니다.

import numpy as np

def ant_colony_optimization(distance_matrix, n_ants, n_iterations, alpha=1.0, beta=2.0, evaporation_rate=0.5, initial_pheromone=1.0):
    n_cities = len(distance_matrix)
    
    # 페로몬 행렬 초기화
    pheromone = np.ones((n_cities, n_cities)) * initial_pheromone
    
    # 최적 경로 및 거리 초기화
    best_path = None
    best_distance = float('inf')
    
    for iteration in range(n_iterations):
        # 모든 개미의 경로 생성
        all_paths = []
        all_distances = []
        
        for ant in range(n_ants):
            # 무작위 시작 도시
            current_city = np.random.randint(n_cities)
            path = [current_city]
            visited = [False] * n_cities
            visited[current_city] = True
            
            # 나머지 도시 방문
            for _ in range(n_cities - 1):
                # 다음 도시 선택 확률 계산
                probabilities = np.zeros(n_cities)
                
                for city in range(n_cities):
                    if not visited[city]:
                        # 페로몬과 휴리스틱 정보 결합
                        pheromone_factor = pheromone[current_city][city] ** alpha
                        heuristic_factor = (1.0 / distance_matrix[current_city][city]) ** beta
                        probabilities[city] = pheromone_factor * heuristic_factor
                
                # 확률에 따라 다음 도시 선택
                probabilities = probabilities / np.sum(probabilities)
                next_city = np.random.choice(range(n_cities), p=probabilities)
                
                path.append(next_city)
                visited[next_city] = True
                current_city = next_city
            
            # 경로 거리 계산
            path_distance = sum(distance_matrix[path[i]][path[i+1]] for i in range(n_cities-1))
            path_distance += distance_matrix[path[-1]][path[0]]  # 출발점으로 돌아오기
            
            all_paths.append(path)
            all_distances.append(path_distance)
            
            # 최적 경로 업데이트
            if path_distance < best_distance:
                best_distance = path_distance
                best_path = path.copy()
        
        # 페로몬 증발
        pheromone *= (1 - evaporation_rate)
        
        # 페로몬 업데이트
        for ant, (path, distance) in enumerate(zip(all_paths, all_distances)):
            pheromone_deposit = 1.0 / distance
            
            for i in range(n_cities - 1):
                pheromone[path[i]][path[i+1]] += pheromone_deposit
                pheromone[path[i+1]][path[i]] += pheromone_deposit  # 양방향 그래프인 경우
            
            # 출발점으로 돌아오는 경로도 업데이트
            pheromone[path[-1]][path[0]] += pheromone_deposit
            pheromone[path[0]][path[-1]] += pheromone_deposit
    
    return best_path, best_distance

ACO의 장점은 그래프 기반 문제(예: 외판원 문제, 차량 라우팅)에 매우 효과적이며, 집단 지성과 양성 피드백 메커니즘을 활용한다는 점입니다.

import numpy as np
import random

class KnapsackProblem:
    def __init__(self, weights, values, capacity):
        self.weights = weights
        self.values = values
        self.capacity = capacity
        self.n_items = len(weights)
    
    def fitness(self, solution):
        total_weight = sum(solution[i] * self.weights[i] for i in range(self.n_items))
        
        # 용량 초과 시 패널티 부여
        if total_weight > self.capacity:
            return 0
        
        total_value = sum(solution[i] * self.values[i] for i in range(self.n_items))
        return total_value

def genetic_algorithm_knapsack(knapsack_problem, population_size=100, generations=500, 
                               crossover_rate=0.8, mutation_rate=0.1):
    # 초기 개체군 생성
    population = [
        [random.randint(0, 1) for _ in range(knapsack_problem.n_items)]
        for _ in range(population_size)
    ]
    
    best_solution = None
    best_fitness = 0
    
    for generation in range(generations):
        # 적합도 평가
        fitness_scores = [knapsack_problem.fitness(ind) for ind in population]
        
        # 최고 해결책 업데이트
        max_fitness = max(fitness_scores)
        if max_fitness > best_fitness:
            best_fitness = max_fitness
            best_solution = population[fitness_scores.index(max_fitness)].copy()
        
        # 새 개체군 생성
        new_population = []
        
        # 엘리트주의: 최고 해결책 보존
        new_population.append(best_solution)
        
        while len(new_population) < population_size:
            # 토너먼트 선택
            parent1 = tournament_selection(population, fitness_scores)
            parent2 = tournament_selection(population, fitness_scores)
            
            # 교차
            if random.random() < crossover_rate:
                crossover_point = random.randint(1, knapsack_problem.n_items - 1)
                child = parent1[:crossover_point] + parent2[crossover_point:]
            else:
                child = parent1.copy()
            
            # 변이
            for i in range(len(child)):
                if random.random() < mutation_rate:
                    child[i] = 1 - child[i]  # 비트 반전
            
            new_population.append(child)
        
        # 개체군 교체
        population = new_population
    
    return best_solution, best_fitness

def tournament_selection(population, fitness_scores, tournament_size=3):
    # 토너먼트 선택: 몇 개의 무작위 개체 중 최고 적합도를 가진 개체 선택
    tournament_indices = random.sample(range(len(population)), tournament_size)
    tournament_fitness = [fitness_scores[i] for i in tournament_indices]
    winner_index = tournament_indices[tournament_fitness.index(max(tournament_fitness))]
    return population[winner_index]

print(f"총 가치: {fitness}")

def parallel_simulated_annealing(initial_solutions, cost_function, neighbor_function, 
                                temperature, cooling_rate, max_iterations, n_processes):
    # 병렬 처리를 위한 함수
    def worker(initial_solution):
        return simulated_annealing(initial_solution, cost_function, neighbor_function, 
                                  temperature, cooling_rate, max_iterations)
    
    # 멀티프로세싱 풀 생성
    with multiprocessing.Pool(processes=n_processes) as pool:
        results = pool.map(worker, initial_solutions)
    
    # 최적 결과 선택
    best_solution, best_cost = min(results, key=lambda x: x[1])
    return best_solution, best_cost

                          local_search_iterations):

    population = initialize_population(population_size, chromosome_length)

        fitness_scores = [fitness_function(ind) for ind in population]
        parents = select_parents(population, fitness_scores)
        offspring = crossover_and_mutate(parents)
        
        # 지역 탐색 개선
        for i in range(len(offspring)):
            offspring[i] = local_search(offspring[i], fitness_function, local_search_iterations)

def local_search(solution, fitness_function, iterations):
    current_solution = solution.copy()
    current_fitness = fitness_function(current_solution)
    
    for _ in range(iterations):
        # 이웃 생성
        neighbor = generate_neighbor(current_solution)
        neighbor_fitness = fitness_function(neighbor)
        
        # 더 좋은 이웃을 찾으면 이동
        if neighbor_fitness > current_fitness:
            current_solution = neighbor
            current_fitness = neighbor_fitness
    
    return current_solution

from sklearn.ensemble import RandomForestClassifier
from sklearn.feature_selection import SelectFromModel
import numpy as np

def genetic_feature_selection(X, y, n_estimators=100, n_generations=50, population_size=30):
    n_features = X.shape[1]
    
    # 초기 개체군 생성 (각 개체는 특성의 선택 여부를 나타내는 이진 배열)
    population = [np.random.randint(0, 2, size=n_features) for _ in range(population_size)]
    
    for generation in range(n_generations):
        # 각 특성 부분집합에 대한 모델 성능 평가
        scores = []
        for individual in population:
            # 최소한 하나의 특성은 선택되도록 함
            if sum(individual) == 0:
                individual[np.random.randint(0, n_features)] = 1
            
            # 선택된 특성만 사용하여 모델 훈련 및 평가
            X_selected = X[:, individual == 1]
            model = RandomForestClassifier(n_estimators=n_estimators, random_state=42)
            
            # 교차 검증 점수 계산
            from sklearn.model_selection import cross_val_score
            score = np.mean(cross_val_score(model, X_selected, y, cv=5))
            scores.append(score)
        
        # 다음 세대 생성
        next_population = []
        
        # 엘리트 보존 (최고 성능의 개체 유지)
        elite_idx = np.argmax(scores)
        next_population.append(population[elite_idx])
        
        # 나머지 개체 생성
        while len(next_population) < population_size:
            # 토너먼트 선택
            parent1 = tournament_selection(population, scores)
            parent2 = tournament_selection(population, scores)
            
            # 교차
            child = crossover(parent1, parent2)
            
            # 변이
            child = mutate(child, mutation_rate=0.1)
            
            next_population.append(child)
        
        # 개체군 업데이트
        population = next_population
    
    # 최적의 특성 부분집합 반환
    best_idx = np.argmax(scores)
    best_feature_mask = population[best_idx]
    
    return best_feature_mask
