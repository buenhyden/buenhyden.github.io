Two Pointer Technique

투 포인터 기법 (Two Pointer Technique)#

투 포인터 기법은 배열이나 문자열과 같은 선형 데이터 구조에서 효율적으로 문제를 해결하기 위한 알고리즘 패러다임이다. 이 기법은 두 개의 포인터를 사용하여 데이터를 순회하면서 시간 복잡도를 줄이는 방법으로, 특히 검색, 정렬, 부분 배열 연산 등에서 자주 활용된다.

투 포인터 기법은 배열, 문자열, 연결 리스트와 같은 선형 데이터 구조에서 효율적인 알고리즘을 설계하기 위한 강력한 도구이다.
이 기법은 중첩된 반복문을 사용하는 O(n²) 해법을 O(n) 시간 복잡도로 최적화할 수 있게 해준다.

투 포인터 기법은 반대 방향, 동일 방향, 빠른/느린 포인터 등 다양한 유형으로 활용할 수 있으며, 두 수의 합, 세 수의 합, 중복 제거, 팰린드롬 확인, 사이클 감지 등 다양한 문제를 해결하는 데 적용할 수 있다.

이 기법을 마스터하기 위해서는 다양한 문제를 풀어보며 각 포인터의 역할, 이동 규칙, 종료 조건을 명확히 이해하는 것이 중요하다.
또한 슬라이딩 윈도우와 같은 관련 기법과의 관계도 이해하면 더 폭넓은 문제 해결 능력을 갖출 수 있다.

투 포인터 기법의 기본 개념#

투 포인터 기법은 단일 반복문 내에서 두 개의 포인터를 조작하여 배열이나 문자열을 탐색한다. 이 두 포인터는 보통 시작점, 끝점, 또는 특정 조건에 따라 움직이면서 문제를 해결한다.
중첩된 반복문을 사용하는 대신 두 개의 포인터로 순회하기 때문에 시간 복잡도를 O(n²)에서 O(n)으로 줄일 수 있다.

투 포인터 기법의 유형#

투 포인터 기법은 크게 세 가지 유형으로 나눌 수 있다:

반대 방향 포인터 (Opposite Direction)#

두 포인터가 배열의 양 끝에서 시작하여 서로를 향해 이동한다.

  • 예: 투 섬 문제(Two Sum), 팰린드롬 확인, 컨테이너에 가장 많은 물 담기

def two_sum(arr, target):
    left = 0
    right = len(arr) - 1
    
    while left < right:
        current_sum = arr[left] + arr[right]
        
        if current_sum == target:
            return [left, right]  # 목표값과 일치하는 두 숫자의 인덱스 반환
        elif current_sum < target:
            left += 1  # 합이 작으면 왼쪽 포인터를 오른쪽으로 이동
        else:
            right -= 1  # 합이 크면 오른쪽 포인터를 왼쪽으로 이동
    
    return [-1, -1]  # 목표값과 일치하는 두 숫자가 없는 경우

# 예시
arr = [1, 3, 4, 5, 7, 11]
target = 9
print(two_sum(arr, target))  # 출력: [2, 3] (4 + 5 = 9)

위 알고리즘의 시간 복잡도는 O(n).
배열이 정렬되어 있기 때문에 두 포인터를 이용해 효율적으로 합이 목표값과 일치하는 두 요소를 찾을 수 있다.

2.2 동일 방향 포인터 (Same Direction)#

두 포인터가 모두 배열의 한쪽 끝(주로 왼쪽)에서 시작하여 같은 방향으로 이동한다.

def remove_duplicates(nums):
    if not nums:
        return 0
    
    # 첫 번째 포인터(slow)는 고유 요소의 위치를 추적
    # 두 번째 포인터(fast)는 배열을 순회
    slow = 0
    
    for fast in range(1, len(nums)):
        # 새로운 고유 요소를 발견하면
        if nums[fast] != nums[slow]:
            slow += 1  # slow 포인터 이동
            nums[slow] = nums[fast]  # 새 고유 요소를 slow 위치에 배치
    
    return slow + 1  # 중복이 제거된 배열의 새 길이 (인덱스 + 1)

# 예시
nums = [1, 1, 2, 2, 3, 4, 4, 5]
new_length = remove_duplicates(nums)
print(new_length)  # 출력: 5
print(nums[:new_length])  # 출력: [1, 2, 3, 4, 5]

이 알고리즘의 시간 복잡도도 O(n)입니다. fast 포인터는 배열을 한 번만 순회하고, slow 포인터는 고유 요소의 위치를 추적합니다.

빠른/느린 포인터 (Fast and Slow)#

두 포인터가 서로 다른 속도로 이동한다.
보통 한 포인터는 한 칸씩, 다른 포인터는 두 칸씩 이동한다.

  • 예: 순환 감지, 중간 요소 찾기, 연결 리스트의 사이클 탐지

class ListNode:
    def __init__(self, val=0, next=None):
        self.val = val
        self.next = next

def has_cycle(head):
    if not head or not head.next:
        return False
    
    # 두 포인터 초기화
    slow = head  # 느린 포인터
    fast = head.next  # 빠른 포인터
    
    # fast가 slow를 따라잡을 때까지 이동
    while slow != fast:
        # fast가 끝에 도달하면 사이클이 없음
        if not fast or not fast.next:
            return False
        
        # 느린 포인터는 한 칸씩, 빠른 포인터는 두 칸씩 이동
        slow = slow.next
        fast = fast.next.next
    
    # 두 포인터가 만나면 사이클이 있음
    return True

이 알고리즘은 플로이드의 토끼와 거북이(Floyd’s Tortoise and Hare) 알고리즘으로도 알려져 있다. 시간 복잡도는 O(n)이고 공간 복잡도는 O(1).

def is_palindrome(s):
    # 알파벳과 숫자만 추출하고 소문자로 변환
    s = ''.join(char.lower() for char in s if char.isalnum())
    
    left, right = 0, len(s) - 1
    
    while left < right:
        if s[left] != s[right]:
            return False
        left += 1
        right -= 1
    
    return True

# 예시
print(is_palindrome("A man, a plan, a canal: Panama"))  # 출력: True
print(is_palindrome("race a car"))  # 출력: False

def three_sum(nums):
    result = []
    nums.sort()  # 배열 정렬
    
    for i in range(len(nums) - 2):
        # 중복된 첫 번째 요소 건너뛰기
        if i > 0 and nums[i] == nums[i-1]:
            continue
        
        left = i + 1
        right = len(nums) - 1
        
        while left < right:
            total = nums[i] + nums[left] + nums[right]
            
            if total < 0:
                left += 1
            elif total > 0:
                right -= 1
            else:
                # 합이 0이면 결과에 추가
                result.append([nums[i], nums[left], nums[right]])
                
                # 중복 건너뛰기
                while left < right and nums[left] == nums[left+1]:
                    left += 1
                while left < right and nums[right] == nums[right-1]:
                    right -= 1
                
                left += 1
                right -= 1
    
    return result

# 예시
nums = [-1, 0, 1, 2, -1, -4]
print(three_sum(nums))  # 출력: [[-1, -1, 2], [-1, 0, 1]]

def merge_sorted_arrays(arr1, arr2):
    result = []
    p1, p2 = 0, 0  # 두 배열을 위한 포인터
    
    # 두 배열을 순회하며 작은 값부터 결과 배열에 추가
    while p1 < len(arr1) and p2 < len(arr2):
        if arr1[p1] <= arr2[p2]:
            result.append(arr1[p1])
            p1 += 1
        else:
            result.append(arr2[p2])
            p2 += 1
    
    # 남은 요소 추가
    while p1 < len(arr1):
        result.append(arr1[p1])
        p1 += 1
    
    while p2 < len(arr2):
        result.append(arr2[p2])
        p2 += 1
    
    return result

# 예시
arr1 = [1, 3, 5, 7]
arr2 = [2, 4, 6, 8, 10]
print(merge_sorted_arrays(arr1, arr2))  # 출력: [1, 2, 3, 4, 5, 6, 7, 8, 10]

투 포인터 기법의 장단점#

장점#

  • 효율성: O(n²)의 시간 복잡도를 O(n)으로 줄일 수 있다.
  • 공간 효율성: 대부분의 경우 O(1)의 추가 공간만 사용.
  • 알고리즘 단순화: 복잡한 문제를 간단한 포인터 조작으로 해결할 수 있다.
  • 유연성: 다양한 문제 유형에 적용할 수 있다.

5.2 단점#

  • 적용 제한: 모든 문제에 적용할 수 있는 것은 아니다. 주로 선형 데이터 구조에 적합.
  • 정렬 필요: 많은 투 포인터 알고리즘은 정렬된 입력을 필요로 한다.
  • 복잡한 경우 처리: 세 개 이상의 포인터가 필요한 경우 구현이 복잡해질 수 있다.

투 포인터 기법 활용을 위한 팁#

  1. 문제 유형 파악: 선형 구조(배열, 문자열, 연결 리스트 등)에 관련된 문제인지 확인.
  2. 포인터 위치 결정: 문제에 따라 포인터의 시작 위치와 이동 방향을 결정.
  3. 종료 조건 설정: 포인터가 언제 멈춰야 하는지 명확히 정의.
  4. 포인터 이동 규칙: 각 포인터가 어떤 조건에서 어떻게 이동할지 결정.
  5. 경계 조건 처리: 빈 배열, 단일 요소 배열 등의 경계 조건을 신중하게 처리.

def trap(height):
    if not height:
        return 0
    
    left, right = 0, len(height) - 1
    left_max, right_max = 0, 0
    total_water = 0
    
    while left < right:
        # 더 낮은 벽을 기준으로 물의 양 계산
        if height[left] < height[right]:
            if height[left] >= left_max:
                left_max = height[left]
            else:
                total_water += left_max - height[left]
            left += 1
        else:
            if height[right] >= right_max:
                right_max = height[right]
            else:
                total_water += right_max - height[right]
            right -= 1
    
    return total_water

# 예시
height = [0, 1, 0, 2, 1, 0, 1, 3, 2, 1, 2, 1]
print(trap(height))  # 출력: 6

이 알고리즘의 시간 복잡도는 O(n)이고, 공간 복잡도는 O(1). 투 포인터 기법을 사용하여 각 위치에서 가둘 수 있는 물의 양을 효율적으로 계산.

def max_area(height):
    left, right = 0, len(height) - 1
    max_water = 0
    
    while left < right:
        # 현재 두 선 사이의 물의 양 계산
        water = min(height[left], height[right]) * (right - left)
        max_water = max(max_water, water)
        
        # 더 낮은 선을 안쪽으로 이동
        if height[left] < height[right]:
            left += 1
        else:
            right -= 1
    
    return max_water

# 예시
height = [1, 8, 6, 2, 5, 4, 8, 3, 7]
print(max_area(height))  # 출력: 49

이 알고리즘의 시간 복잡도도 O(n)이고, 공간 복잡도는 O(1).
반대 방향 투 포인터 기법을 사용하여 가능한 모든 컨테이너 중 최대 용량을 효율적으로 찾는다.

실전 문제 해결 전략#

투 포인터 기법을 효과적으로 활용하기 위한 문제 해결 전략:

  1. 문제 분석: 문제가 선형 데이터 구조에 관련되어 있는가?
    • 배열, 문자열, 연결 리스트 등의 선형 구조에서 탐색, 정렬, 부분 집합 찾기 등의 문제인지 확인한다.
  2. 포인터 유형 결정: 어떤 유형의 투 포인터가 적합한가?
    • 반대 방향 포인터: 두 요소의 조합을 찾는 문제 (예: 두 수의 합)
    • 동일 방향 포인터: 부분 배열이나 윈도우 처리 문제 (예: 중복 제거)
    • 빠른/느린 포인터: 사이클 감지나 중간 요소 찾기 문제 (예: 연결 리스트의 사이클)
  3. 포인터 초기화 및 이동 규칙 설정:
    • 포인터의 시작 위치를 어디로 할 것인가? (배열의 시작, 끝, 중간 등)
    • 각 포인터는 어떤 조건에서 어떻게 이동할 것인가?
    • 포인터 이동이 문제의 특성을 어떻게 활용하는가?
  4. 종료 조건 설정:
    • 포인터가 서로 만나거나 교차할 때?
    • 특정 조건이 충족될 때?
    • 모든 요소를 순회했을 때?
  5. 결과 업데이트 조건:
    • 어떤 조건에서 최종 결과를 업데이트할 것인가?
    • 현재 포인터 위치에서의 계산 결과를 어떻게 활용할 것인가?

Two Pointer Technique vs. Sliding Window Technique

투 포인터(Two Pointer)와 슬라이딩 윈도우(Sliding Window) 투 포인터와 슬라이딩 윈도우 기법은 모두 배열이나 문자열과 같은 선형 데이터 구조에서 효율적인 탐색과 연산을 가능하게 하는 강력한 알고리즘 패러다임이다. 이 두 기법은 중첩된 반복문을 사용하는 O(n²) 해법을 O(n) 시간 복잡도로 최적화할 수 있게 해주어, 성능이 중요한 애플리케이션에서 필수적인 도구가 된다. 투 포인터 기법은 더 일반적이고 유연한 접근 방식으로, 두 포인터가 독립적으로 이동할 수 있어 다양한 문제 유형에 적용 가능하다. 반면, 슬라이딩 윈도우 기법은 연속된 요소들의 집합을 효율적으로 처리하는 데 특화되어 있으며, 투 포인터의 특수한 형태로 볼 수 있다. ...
