Hash Map#

HashMap은 키-값 쌍을 저장하고 관리하는 연관 배열의 구현체로, 효율적인 검색, 삽입, 삭제 연산을 제공한다.

HashMap은 해시 함수를 사용하여 키를 배열의 인덱스로 변환하고, 해당 인덱스에 값을 저장하는 데이터 구조이다.
이는 키를 통해 빠르게 값을 검색할 수 있게 해준다.

특징#

  1. 키-값 쌍 저장: 각 데이터는 고유한 키와 연관된 값으로 저장된다.
  2. 빠른 검색: 평균적으로 O(1) 시간 복잡도로 데이터를 검색할 수 있다.
  3. 동적 크기 조정: 데이터 양에 따라 자동으로 크기를 조정한다.
  4. 널(null) 허용: 대부분의 구현에서 널 키와 널 값을 허용한다.

장점#

  1. 빠른 접근 및 수정: 키를 통한 빠른 데이터 접근과 수정이 가능하다.
  2. 유연성: 다양한 타입의 키와 값을 저장할 수 있다.
  3. 메모리 효율성: 데이터 양에 따라 동적으로 크기를 조절한다.

단점#

  1. 충돌 가능성: 서로 다른 키가 같은 해시 값을 가질 수 있어 충돌이 발생할 수 있다.
  2. 순서 보장 없음: 데이터의 삽입 순서가 보장되지 않는다.
  3. 메모리 오버헤드: 해시 테이블 구조로 인한 추가적인 메모리 사용이 있을 수 있다.

응용#

  1. 데이터베이스 인덱싱
  2. 캐시 구현
  3. 심볼 테이블 관리
  4. 빠른 데이터 검색이 필요한 다양한 애플리케이션

동작 원리#

  1. 해시 함수: 키를 입력받아 배열의 인덱스로 변환한다.
  2. 충돌 해결: 서로 다른 키가 같은 인덱스를 가리킬 때 충돌을 해결하는 방법을 사용한다.

구성 요소#

  1. 버킷: 실제 데이터를 저장하는 배열의 각 요소.
  2. 키(Key): 데이터를 식별하는 고유한 값.
  3. 값(Value): 키와 연관된 실제 데이터.
  4. 해시 함수: 키를 해시 코드로 변환하는 함수.

구현 방식#

일반적으로 배열과 연결 리스트를 조합하여 구현한다.
배열은 버킷을 저장하고, 각 버킷은 연결 리스트로 충돌을 해결한다.

주요 연산들의 동작 과정#

  1. 삽입(put):

    • 키의 해시 값을 계산한다.
    • 해시 값을 인덱스로 변환하여 해당 버킷을 찾는다.
    • 버킷에 키-값 쌍을 저장합니다. 충돌 시 연결 리스트에 추가한다.

  2. 검색(get):

    • 키의 해시 값을 계산한다.
    • 해당 버킷을 찾아 키와 일치하는 항목을 반환한다.

  3. 삭제(remove):

    • 키의 해시 값을 계산하여 버킷을 찾는다.
    • 해당 키-값 쌍을 제거한다.

import java.util.HashMap;

public class HashMapExample {
    public static void main(String[] args) {
        HashMap<String, Integer> map = new HashMap<>();
        
        // 삽입
        map.put("apple", 1);
        map.put("banana", 2);
        map.put("cherry", 3);
        
        // 검색
        System.out.println(map.get("banana")); // 출력: 2
        
        // 삭제
        map.remove("apple");
        
        // 순회
        for (String key : map.keySet()) {
            System.out.println(key + ": " + map.get(key));
        }
    }
}
