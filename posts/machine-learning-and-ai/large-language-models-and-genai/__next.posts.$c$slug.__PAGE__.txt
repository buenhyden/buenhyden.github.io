## 1. 개요 (Overview)

`거대 언어 모델 및 생성형 AI(Large Language Models & GenAI)`는 수조 개의 데이터 패킷(Tokens) 사이의 통계적 인력을 활용하여 복잡한 맥락을 추론하고 새로운 지능적 결과물을 생성하는 **확률적 생성 공학**입니다. 단순한 패턴 매칭을 넘어, 모델 아키텍처가 임계점(Scale)을 넘었을 때 발현되는 창발적 능력의 물리적 증거를 탐구하며, 지능의 출력을 비즈니스 목적에 맞게 정밀하게 타격(Alignment)하는 고도의 제어 기술을 다룹니다.

## 2. 범위 및 경계 (Scope & Boundaries)

- **포함 범위 (In-Scope)**:
  - **LLM 코어 물리**: Scaling Laws(크기의 법칙), 토큰 임베딩 사영 및 다음 토큰 예측 역학.
  - **검색 및 맥락 제어**: RAG(Retrieval-Augmented Generation)의 벡터 검색 물리 및 컨텍스트 윈도우 관리.
  - **지능의 튜닝**: PEFT(LoRA 등)의 가중치 미세 조정 물리 및 RLHF를 통한 인간 가치 정렬.
  - **에이전트 및 멀티모달**: 추론 루프(Reasoning Loop) 설계 및 이미지·음성 등 이종 데이터 간의 잠재 공간 통합.
- **제외 범위 (Out-of-Scope)**:
  - 트랜스포머 레이어의 구체적인 수치 미분 연쇄 법칙 ([Neural Networks](/posts/machine-learning-and-ai/deep-learning-and-neural-networks/) 담당).
  - 지능 시스템의 안전한 배포를 위한 쿠버네티스 인프라 상세 ([MLOps Dynamics](/posts/machine-learning-and-ai/mlops-and-production-ai-dynamics/) 담당).
- **경계 근거**: CS2023 AI-GenAI Area, Industry LLM Standards (OpenAI, Anthropic).

## 3. 반례 (Counterexample)

- **'LLM = Search Engine' 오해**: LLM은 백과사전식 정보를 가져오는 검색 엔진이 아닙니다. LLM은 수집된 지식의 '확률적 분포'를 재구성하여 답변을 생성하는 기계이며, 외부 실시간 정보와의 물리적 연결(RAG) 없이는 최신성이나 사실 정확도를 보장할 수 없습니다.
- **'Generation = Creativity Without Logic' 오해**: AI의 생성은 순수한 무작위 창작이 아닙니다. 모든 출력은 모델 내부의 잠재 공간(Latent Space)에 선언된 물리적 좌표와 온도(Temperature) 변수에 의한 엄격한 수치적 샘플링의 결과이며, 이를 논리적으로 제어하는 것이 생성 AI 공학의 본질입니다.

## 4. 선행 지식 (Prerequisites)

- **Pillar 08. Network & Communication**: 대규모 데이터 통신을 위한 API 연동 및 스트리밍(Streaming) 데이터 처리 이해.
- **Node 11. 02 Deep Learning & Neural Networks**: 트랜스포머 아키텍처의 셀프 어텐션(Self-Attention) 작동 원리 숙지.

## 5. 학습 지도 (Learning Map)

1. **토큰의 질량(Foundations)**: 낱말 하나하나가 어떻게 고차원 벡터로 변환되고, 의미의 중력에 따라 서로를 끌어당기는지 물리적 배치를 배웁니다.
2. **지능의 조준(Prompting)**: 모델에게 특정 질문을 던져 방대한 잠재 공간 중 우리가 원하는 지점만을 정확히 타격하도록 유도하는 기술을 익힙니다.
3. **지능의 확장(RAG)**: 모델의 뇌 용량 한계를 넘어, 외부 도서관(Vector DB)과 물리적으로 연결하여 지능의 정확도를 높이는 아키텍처를 파악합니다.
4. **윤리의 중력(Alignment)**: 기계의 거침없는 예측이 인간의 규범과 충돌하지 않도록, 가치관의 물리적 가드레일을 설치하는 법을 습득합니다.

## 7. 단계별 학습 토픽 (Learning Topics)

### 기본 (Basic)

- **Next-Token Prediction Logic**: 현재까지의 문맥을 바탕으로 가장 높은 확률의 물리적 토큰을 골라내는 LLM의 근본 작동 원리.
- **Scaling Laws for Neural Models**: 데이터양과 연산량이 증가함에 따라 모델 성능이 지수적으로 개선되는 물리적 임계점 분석.

### 권장 (Recommended)

- **Vector Database Mechanics**: 텍스트를 기하학적 좌표로 변환하고, 유사한 의미를 가진 정보를 물리적으로 빠르게 찾아내는 검색 물리.
- **Temperature & Sampling Constraints**: 생성의 창의성(Entropy)과 사실성 사이의 수치적 균형을 조절하는 확률 제어 기술.

### 실무 (Practical)

- **RAG System Architecture**: 문서 추출, 청킹(Chunking), 임베딩, 검색, 생성으로 이어지는 실제 지능 보강 파이프라인 구축 실무.
- **Low-Rank Adaptation (LoRA)**: 모델 전체를 바꾸지 않고 극히 일부의 파라미터만 비틀어 지능의 방향을 수정하는 효율적 물리 튜닝.

### 심화 (Advanced)

-   **RLHF & DPO Dynamics**: 인간의 선호를 보상 함수로 구성하여 모델의 논리 흐름을 물리적으로 강제 정렬하는 기술.
-   **Agentic Reasoning Loops**: 모델이 스스로 도구를 사용하고(Task Tools) 결과를 성찰하며 목표를 달성해 나가는 자율 에이전트 구축 역학.

### 실습 및 참조 구현 (Hands-on Labs)

-   **Lab 01**: Hugging Face의 Transformers 라이브러리를 사용하여 사전 학습된 모델에 LoRA 미세 조정을 적용하고, 특정 도메인에 대한 답변 정확도의 물리적 변화를 측정하는 실습.
-   **Lab 02**: LangChain이나 LlamaIndex를 사용하여 벡터 데이터베이스를 연동한 RAG 파이프라인을 구축하고, '검색된 지식'이 모델의 출력 신뢰도에 미치는 물리적 영향을 분석하는 실습.

15:T3081,<h2 id="1-overview">1. 개요 (Overview)</h2>
<p><code>거대 언어 모델 및 생성형 AI(Large Language Models &#x26; GenAI)</code>는 수조 개의 데이터 패킷(Tokens) 사이의 통계적 인력을 활용하여 복잡한 맥락을 추론하고 새로운 지능적 결과물을 생성하는 <strong>확률적 생성 공학</strong>입니다. 단순한 패턴 매칭을 넘어, 모델 아키텍처가 임계점(Scale)을 넘었을 때 발현되는 창발적 능력의 물리적 증거를 탐구하며, 지능의 출력을 비즈니스 목적에 맞게 정밀하게 타격(Alignment)하는 고도의 제어 기술을 다룹니다.</p>
<h2 id="2-scope-boundaries">2. 범위 및 경계 (Scope &#x26; Boundaries)</h2>
<ul>
<li><strong>포함 범위 (In-Scope)</strong>:
<ul>
<li><strong>LLM 코어 물리</strong>: Scaling Laws(크기의 법칙), 토큰 임베딩 사영 및 다음 토큰 예측 역학.</li>
<li><strong>검색 및 맥락 제어</strong>: RAG(Retrieval-Augmented Generation)의 벡터 검색 물리 및 컨텍스트 윈도우 관리.</li>
<li><strong>지능의 튜닝</strong>: PEFT(LoRA 등)의 가중치 미세 조정 물리 및 RLHF를 통한 인간 가치 정렬.</li>
<li><strong>에이전트 및 멀티모달</strong>: 추론 루프(Reasoning Loop) 설계 및 이미지·음성 등 이종 데이터 간의 잠재 공간 통합.</li>
</ul>
</li>
<li><strong>제외 범위 (Out-of-Scope)</strong>:
<ul>
<li>트랜스포머 레이어의 구체적인 수치 미분 연쇄 법칙 (<a href="/posts/machine-learning-and-ai/deep-learning-and-neural-networks/">Neural Networks</a> 담당).</li>
<li>지능 시스템의 안전한 배포를 위한 쿠버네티스 인프라 상세 (<a href="/posts/machine-learning-and-ai/mlops-and-production-ai-dynamics/">MLOps Dynamics</a> 담당).</li>
</ul>
</li>
<li><strong>경계 근거</strong>: CS2023 AI-GenAI Area, Industry LLM Standards (OpenAI, Anthropic).</li>
</ul>
<h2 id="3-counterexample">3. 반례 (Counterexample)</h2>
<ul>
<li><strong>'LLM = Search Engine' 오해</strong>: LLM은 백과사전식 정보를 가져오는 검색 엔진이 아닙니다. LLM은 수집된 지식의 '확률적 분포'를 재구성하여 답변을 생성하는 기계이며, 외부 실시간 정보와의 물리적 연결(RAG) 없이는 최신성이나 사실 정확도를 보장할 수 없습니다.</li>
<li><strong>'Generation = Creativity Without Logic' 오해</strong>: AI의 생성은 순수한 무작위 창작이 아닙니다. 모든 출력은 모델 내부의 잠재 공간(Latent Space)에 선언된 물리적 좌표와 온도(Temperature) 변수에 의한 엄격한 수치적 샘플링의 결과이며, 이를 논리적으로 제어하는 것이 생성 AI 공학의 본질입니다.</li>
</ul>

<h2 id="5-learning-map">5. 학습 지도 (Learning Map)</h2>
<ol>
<li><strong>토큰의 질량(Foundations)</strong>: 낱말 하나하나가 어떻게 고차원 벡터로 변환되고, 의미의 중력에 따라 서로를 끌어당기는지 물리적 배치를 배웁니다.</li>
<li><strong>지능의 조준(Prompting)</strong>: 모델에게 특정 질문을 던져 방대한 잠재 공간 중 우리가 원하는 지점만을 정확히 타격하도록 유도하는 기술을 익힙니다.</li>
<li><strong>지능의 확장(RAG)</strong>: 모델의 뇌 용량 한계를 넘어, 외부 도서관(Vector DB)과 물리적으로 연결하여 지능의 정확도를 높이는 아키텍처를 파악합니다.</li>
<li><strong>윤리의 중력(Alignment)</strong>: 기계의 거침없는 예측이 인간의 규범과 충돌하지 않도록, 가치관의 물리적 가드레일을 설치하는 법을 습득합니다.</li>
</ol>

<h2 id="7-learning-topics">7. 단계별 학습 토픽 (Learning Topics)</h2>
<h3 id="basic">기본 (Basic)</h3>
<ul>
<li><strong>Next-Token Prediction Logic</strong>: 현재까지의 문맥을 바탕으로 가장 높은 확률의 물리적 토큰을 골라내는 LLM의 근본 작동 원리.</li>
<li><strong>Scaling Laws for Neural Models</strong>: 데이터양과 연산량이 증가함에 따라 모델 성능이 지수적으로 개선되는 물리적 임계점 분석.</li>
</ul>
<h3 id="recommended">권장 (Recommended)</h3>
<ul>
<li><strong>Vector Database Mechanics</strong>: 텍스트를 기하학적 좌표로 변환하고, 유사한 의미를 가진 정보를 물리적으로 빠르게 찾아내는 검색 물리.</li>
<li><strong>Temperature &#x26; Sampling Constraints</strong>: 생성의 창의성(Entropy)과 사실성 사이의 수치적 균형을 조절하는 확률 제어 기술.</li>
</ul>
<h3 id="practical">실무 (Practical)</h3>
<ul>
<li><strong>RAG System Architecture</strong>: 문서 추출, 청킹(Chunking), 임베딩, 검색, 생성으로 이어지는 실제 지능 보강 파이프라인 구축 실무.</li>
<li><strong>Low-Rank Adaptation (LoRA)</strong>: 모델 전체를 바꾸지 않고 극히 일부의 파라미터만 비틀어 지능의 방향을 수정하는 효율적 물리 튜닝.</li>
</ul>
<h3 id="advanced">심화 (Advanced)</h3>
<ul>
<li><strong>RLHF &#x26; DPO Dynamics</strong>: 인간의 선호를 보상 함수로 구성하여 모델의 논리 흐름을 물리적으로 강제 정렬하는 기술.</li>
<li><strong>Agentic Reasoning Loops</strong>: 모델이 스스로 도구를 사용하고(Task Tools) 결과를 성찰하며 목표를 달성해 나가는 자율 에이전트 구축 역학.</li>
</ul>
<h3 id="hands-on-labs">실습 및 참조 구현 (Hands-on Labs)</h3>
<ul>
<li><strong>Lab 01</strong>: Hugging Face의 Transformers 라이브러리를 사용하여 사전 학습된 모델에 LoRA 미세 조정을 적용하고, 특정 도메인에 대한 답변 정확도의 물리적 변화를 측정하는 실습.</li>
<li><strong>Lab 02</strong>: LangChain이나 LlamaIndex를 사용하여 벡터 데이터베이스를 연동한 RAG 파이프라인을 구축하고, '검색된 지식'이 모델의 출력 신뢰도에 미치는 물리적 영향을 분석하는 실습.</li>
</ul>
