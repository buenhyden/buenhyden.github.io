Jenkins#

특징#

  1. 오픈소스: 무료로 사용 가능하며 커뮤니티의 지원을 받습니다.
  2. 플러그인 생태계: 다양한 플러그인을 통해 기능을 확장할 수 있습니다.
  3. 분산 빌드: 여러 머신에서 빌드 작업을 분산하여 처리할 수 있습니다.
  4. 크로스 플랫폼: Windows, Linux, macOS 등 다양한 운영 체제에서 실행 가능합니다.
  5. 웹 인터페이스: 사용하기 쉬운 웹 기반 인터페이스를 제공합니다.

주요 기능#

  1. 자동화된 빌드 및 테스트
  2. 지속적 통합 및 지속적 배포(CI/CD)
  3. 코드 품질 분석
  4. 배포 자동화
  5. 모니터링 및 알림

구성 요소#

  1. Jenkins 서버: 중앙 제어 서버로 작업을 조정합니다.
  2. 노드(Agents): 실제 작업을 수행하는 워커 머신입니다.
  3. 잡(Jobs): 수행할 작업의 단위입니다.
  4. 플러그인: Jenkins의 기능을 확장하는 추가 모듈입니다.
  5. 파이프라인: 작업의 흐름을 정의하는 스크립트입니다.

장점#

  1. 유연성과 확장성
  2. 광범위한 플러그인 지원
  3. 활발한 커뮤니티 지원
  4. 무료 오픈소스

단점#

  1. 초기 설정이 복잡할 수 있음
  2. 학습 곡선이 가파름
  3. 보안 설정에 주의가 필요함
  4. 리소스 소비가 높을 수 있음

Jenkins 설정 방법#

  1. Jenkins 설치
  2. 초기 설정 및 플러그인 설치
  3. 보안 설정
  4. 노드 구성
  5. 잡 생성 및 파이프라인 설정

pipeline {
    agent any // 어떤 Jenkins 에이전트에서도 실행 가능하도록 설정
    
	// 환경변수 설정
    environment {
        DOCKER_IMAGE = 'my-app:latest'
    }
    
    // 실행 단계 정의
    stages {
	    // 소스 코드 체크아웃 단계
        stage('Checkout') { 
            steps {
	            // GitHub 저장소에서 코드 가져오기
                git 'https://github.com/your-repo/your-project.git' 
            }
        }
        // maven 빌드 단계
        stage('Build') { 
            steps {
	            // Maven을 사용하여 프로젝트 빌드
                sh 'mvn clean package' 
            }
        }
        
        // 테스트 단계
        stage('Test') { 
            steps {
	            // Maven을 사용하여 단위 테스트 실행
                sh 'mvn test' 
            }            
        }
        
		// Docker 이미지 빌드 단계
        stage('Docker Build') { 
            steps {
	            // Dockerfile을 사용하여 Docker 이미지 생성
                sh 'docker build -t your-image:${BUILD_NUMBER} .' 
            }
        }
        
        // 배포 단계
        stage('Deploy') { 
            steps {
	            // 생성된 Docker 이미지를 레지스트리에 푸시
                sh 'docker push your-registry/your-image:${BUILD_NUMBER}' 
                // Kubernetes에 새 이미지 배포
                sh 'kubectl set image deployment/your-app your-container=your-registry/your-image:${BUILD_NUMBER}' 
            }
        }
    }
    
    // 파이프라인 실행 후 작업
    post { 
        always {
            junit '**/target/surefire-reports/*.xml' // 테스트 결과 보고서 생성
        }
        success {
            // 성공 시 슬랙 알림
            slackSend channel: '#deploy',
                      color: 'good',
                      message: "배포 성공: ${env.JOB_NAME} ${env.BUILD_NUMBER}"
        }
        failure {
            // 실패 시 슬랙 알림
            slackSend channel: '#deploy',
                      color: 'danger',
                      message: "배포 실패: ${env.JOB_NAME} ${env.BUILD_NUMBER}"
        }
    }
}
  1. Checkout: GitHub에서 소스 코드를 가져옵니다.
  2. Build: Maven을 사용하여 프로젝트를 빌드합니다.
  3. Test: 단위 테스트를 실행합니다.
  4. Docker Build: 애플리케이션의 Docker 이미지를 생성합니다.
  5. Deploy: 생성된 Docker 이미지를 레지스트리에 푸시하고 Kubernetes에 배포합니다.
