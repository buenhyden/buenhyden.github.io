로드밸런싱 (Load Balancing)#

여러 서버나 네트워크 리소스에 작업량을 분산시켜주는 방법으로 이를 통해 특정 서버에 부하가 집중되는 것을 방지하고, 전체 시스템의 안정성과 가용성을 높일 수 있다.

개념#

로드 밸런싱은 ‘부하 분산’이라고도 불리며, 다음과 같은 핵심 개념을 포함한다:

  1. 트래픽 분산: 클라이언트로부터 오는 네트워크 요청을 여러 서버에 균형있게 배분합니다.
  2. 서버 풀 관리: 여러 대의 서버를 하나의 그룹으로 관리하여 효율적으로 리소스를 활용합니다.
  3. 가용성 향상: 특정 서버에 장애가 발생해도 다른 서버로 트래픽을 리디렉션하여 서비스의 연속성을 보장합니다.

주요 특징#

  1. 가용성(Availability) 보장 시스템의 일부가 실패하더라도 서비스를 계속 제공할 수 있습니다. 예를 들어, 3대의 서버 중 1대가 다운되어도 나머지 2대의 서버로 서비스를 계속할 수 있습니다.
  2. 확장성(Scalability) 제공 트래픽이 증가할 때 서버를 추가하기만 하면 됩니다. 마치 음식점에서 손님이 많아지면 웨이터를 추가로 투입하는 것과 같습니다.
  3. 유연성(Flexibility) 확보 서버 유지보수나 업데이트를 할 때, 특정 서버를 작업하는 동안 다른 서버로 트래픽을 우회시킬 수 있습니다.

장점#

  1. 서비스 가용성 향상: 단일 서버 장애 시에도 서비스 중단을 방지합니다.
  2. 성능 최적화: 전체 시스템의 처리 능력과 응답 시간을 개선합니다.
  3. 확장성 제공: 트래픽 증가에 따라 유연하게 서버를 추가할 수 있습니다.
  4. 보안 강화: L7 로드 밸런서의 경우 애플리케이션 레벨에서의 보안 기능을 제공할 수 있습니다.

단점#

  1. 초기 설정 복잡성: 로드 밸런서 구성과 관리에 전문 지식이 필요할 수 있습니다.
  2. 추가 비용: 하드웨어 로드 밸런서의 경우 구매 및 유지보수 비용이 발생합니다.
  3. 단일 실패 지점: 로드 밸런서 자체가 장애 지점이 될 수 있어 이중화 구성이 필요할 수 있습니다.

종류#

  1. L4 로드 밸런싱:

    • 전송 계층(TCP/UDP)에서 작동합니다.
    • IP 주소와 포트 정보를 기반으로 트래픽을 분산합니다.
    • 빠른 처리 속도와 효율성이 특징입니다.

  2. L7 로드 밸런싱:

    • 애플리케이션 계층에서 작동합니다.
    • HTTP 헤더, 쿠키 등 애플리케이션 레벨의 정보를 활용하여 더 섬세한 라우팅이 가능합니다.
    • 콘텐츠 기반 라우팅과 보안 기능을 제공할 수 있습니다.

    class RoundRobinLoadBalancer:
    def __init__(self, servers):
        self.servers = servers  # 서버 목록
        self.current = 0  # 현재 서버 인덱스

    def get_next_server(self):
        server = self.servers[self.current]
        self.current = (self.current + 1) % len(self.servers)
        return server

  2. 가중 라운드 로빈: 서버의 처리 능력에 따라 가중치를 부여하여 분배합니다.

    class WeightedLoadBalancer:
    def __init__(self, servers_with_weights):
        self.servers = servers_with_weights  # (서버, 가중치) 튜플 리스트

    def get_next_server(self):
        total_weight = sum(weight for _, weight in self.servers)
        r = random.uniform(0, total_weight)
        for server, weight in self.servers:
            if r <= weight:
                return server
            r -= weight

    class LeastConnectionLoadBalancer:
    def __init__(self, servers):
        self.servers = {server: 0 for server in servers}  # 서버별 현재 연결 수

    def get_next_server(self):
        return min(self.servers.items(), key=lambda x: x[1])[0]

  4. IP 해시: 클라이언트의 IP 주소를 기반으로 특정 서버에 요청을 매핑합니다.
