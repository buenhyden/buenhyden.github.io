Docker#

애플리케이션과 그 실행 환경을 컨테이너라는 독립된 단위로 패키징하여 어디서나 일관되게 실행할 수 있게 해주는 플랫폼.
컨테이너는 애플리케이션 코드, 런타임, 시스템 도구, 라이브러리 등 필요한 모든 것을 포함하는 경량화된 가상화 환경.

특징#

  1. 포터빌리티: Docker 컨테이너는 어떤 환경에서도 동일하게 실행될 수 있어 개발, 테스트, 운영 환경 간 일관성을 제공합니다.
  2. 경량화: 컨테이너는 호스트 OS의 커널을 공유하여 가상 머신보다 더 가볍고 효율적입니다.
  3. 격리: 각 컨테이너는 독립적으로 실행되어 다른 컨테이너나 호스트 시스템에 영향을 주지 않습니다.
  4. 버전 관리: Docker 이미지는 버전 관리가 가능하며, Docker Hub를 통해 쉽게 공유할 수 있습니다.

장점#

  1. 일관된 개발 환경: 개발자들이 동일한 환경에서 작업할 수 있어 “내 컴퓨터에서는 작동합니다” 문제를 해결합니다.
  2. 빠른 배포: 컨테이너화된 애플리케이션은 빠르게 배포되고 확장될 수 있습니다.
  3. 리소스 효율성: 컨테이너는 가상 머신보다 적은 리소스를 사용합니다.
  4. 마이크로서비스 아키텍처 지원: Docker는 독립적인 서비스 구현에 이상적입니다.

단점#

  1. 보안 이슈: 컨테이너 간 격리가 완벽하지 않아 보안 취약점이 발생할 수 있습니다.
  2. 복잡성: Docker의 학습 곡선이 가파를 수 있으며, 관리가 복잡해질 수 있습니다.
  3. 제한된 오케스트레이션: 다중 컨테이너 관리를 위해서는 추가 도구가 필요할 수 있습니다.

역할#

  1. 개발 환경 표준화: 모든 개발자가 동일한 환경에서 작업할 수 있게 합니다.
  2. 애플리케이션 격리: 각 애플리케이션을 독립적인 환경에서 실행합니다.
  3. CI/CD 파이프라인 지원: 지속적 통합 및 배포 프로세스를 간소화합니다.
  4. 마이크로서비스 구현: 복잡한 애플리케이션을 작은 독립 서비스로 분리하는 데 도움을 줍니다.

주요 명령어#

  1. 이미지 관리:

    • docker pull <이미지명>: Docker Hub에서 이미지 다운로드
    • docker images: 로컬 이미지 목록 표시
    • docker rmi <이미지명>: 이미지 삭제

  2. 컨테이너 관리:

    • docker run <옵션> <이미지명>: 새 컨테이너 생성 및 실행
      예: docker run -d --name my_container ubuntu
    • docker ps: 실행 중인 컨테이너 목록 표시
    • docker stop <컨테이너명>: 컨테이너 중지
    • docker start <컨테이너명>: 중지된 컨테이너 시작
    • docker rm <컨테이너명>: 컨테이너 삭제

  3. 실행 중인 컨테이너 조작:

    • docker exec -it <컨테이너명> <명령어>: 실행 중인 컨테이너에 명령 실행
      예: docker exec -it my_container bash

  4. 로그 및 정보:

    • docker logs <컨테이너명>: 컨테이너 로그 확인
    • docker inspect <컨테이너명>: 컨테이너 상세 정보 확인

  5. 네트워크 및 볼륨:

    • docker network create <네트워크명>: 새 Docker 네트워크 생성
    • docker volume create <볼륨명>: 새 Docker 볼륨 생성
