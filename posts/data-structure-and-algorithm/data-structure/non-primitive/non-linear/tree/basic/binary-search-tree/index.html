이진 검색 트리 (Binary Search Tree)#

BST는 계층적 데이터를 정렬된 방식으로 저장하고 조직하는 데 사용되는 특수한 이진 트리 구조이다.
데이터베이스, 검색 엔진, 파일 시스템 등 다양한 응용 프로그램에서 중요한 도구로 사용된다.

이진 검색 트리는 다음과 같은 속성을 가진 이진 트리이다:

  1. 각 노드의 왼쪽 서브트리에는 해당 노드의 값보다 작은 값들만 포함된다.
  2. 각 노드의 오른쪽 서브트리에는 해당 노드의 값보다 큰 값들만 포함된다.
  3. 왼쪽과 오른쪽 서브트리도 각각 이진 검색 트리여야 한다.

다만 이진 검색 트리는 데이터가 정렬된 순서로 입력될 경우 편향된(skewed) 트리가 될 수 있다는 단점이 있다.
이를 해결하기 위해 AVL 트리나 Red-Black 트리와 같은 균형 이진 검색 트리가 개발되었다.

특징#

  1. 정렬된 데이터 저장: BST는 데이터를 정렬된 상태로 유지한다.
  2. 효율적인 검색: 평균적으로 O(log n) 시간 복잡도로 검색이 가능하다.
  3. 동적 구조: 삽입과 삭제가 효율적으로 수행된다.

장점#

  1. 효율적인 검색, 삽입, 삭제 연산: 평균적으로 O(log n) 시간 복잡도.
  2. 정렬된 데이터 유지: 중위 순회를 통해 정렬된 순서로 데이터에 접근 가능.
  3. 범위 검색에 유리: 특정 범위의 값을 쉽게 찾을 수 있음.

응용#

  1. 데이터베이스 인덱싱
  2. 파일 시스템 구현
  3. 우선순위 큐 구현
  4. 심볼 테이블 관리

동작 원리#

  • 검색 연산:
    1. 루트 노드에서 시작한다.
    2. 찾고자 하는 값과 현재 노드의 값을 비교한다.
    3. 찾는 값이 더 작으면 왼쪽 서브트리로, 더 크면 오른쪽 서브트리로 이동한다.
    4. 값을 찾거나 리프 노드에 도달할 때까지 반복한다.
  • 삽입 연산:
    1. 검색과 같은 방식으로 삽입할 위치를 찾는다.
    2. 적절한 위치에 새로운 노드를 추가한다.
  • 삭제 연산:
    1. 삭제할 노드를 찾는다.
    2. 노드의 자식 개수에 따라 다르게 처리한다:
      • 자식이 없는 경우: 직접 삭제
      • 자식이 하나인 경우: 자식으로 대체
      • 자식이 둘인 경우: 후계자(successor)로 대체 후 삭제

구성 요소#

  1. 노드: 키 값과 왼쪽, 오른쪽 자식에 대한 참조를 포함.
  2. 루트: 트리의 최상위 노드.
  3. 리프: 자식이 없는 노드.

class Node:
    def __init__(self, key):
        self.key = key        # 노드의 값
        self.left = None      # 왼쪽 자식 노드
        self.right = None     # 오른쪽 자식 노드

class BinarySearchTree:
    def __init__(self):
        self.root = None      # 트리의 루트 노드

    def insert(self, key):
        # 새로운 키를 삽입하는 메소드
        if not self.root:
            self.root = Node(key)
        else:
            self._insert_recursive(self.root, key)

    def _insert_recursive(self, node, key):
        # 재귀적으로 삽입 위치를 찾아 새 노드를 추가
        if key < node.key:
            if node.left is None:
                node.left = Node(key)
            else:
                self._insert_recursive(node.left, key)
        else:
            if node.right is None:
                node.right = Node(key)
            else:
                self._insert_recursive(node.right, key)

    def search(self, key):
        # 특정 키를 검색하는 메소드
        return self._search_recursive(self.root, key)

    def _search_recursive(self, node, key):
        # 재귀적으로 키를 검색
        if node is None or node.key == key:
            return node

        if key < node.key:
            return self._search_recursive(node.left, key)
        return self._search_recursive(node.right, key)

    def inorder_traversal(self):
        # 중위 순회를 수행하는 메소드
        result = []
        self._inorder_recursive(self.root, result)
        return result

    def _inorder_recursive(self, node, result):
        # 재귀적으로 중위 순회 수행
        if node:
            self._inorder_recursive(node.left, result)
            result.append(node.key)
            self._inorder_recursive(node.right, result)

    def delete(self, key):
        # 특정 키를 삭제하는 메소드
        self.root = self._delete_recursive(self.root, key)

    def _delete_recursive(self, node, key):
        # 재귀적으로 삭제할 노드를 찾고 삭제 수행
        if node is None:
            return node

        if key < node.key:
            node.left = self._delete_recursive(node.left, key)
        elif key > node.key:
            node.right = self._delete_recursive(node.right, key)
        else:
            # 자식이 하나이거나 없는 경우
            if node.left is None:
                return node.right
            elif node.right is None:
                return node.left
            
            # 두 자식이 있는 경우
            # 오른쪽 서브트리에서 최소값을 찾아 현재 노드를 대체
            temp = self._find_min(node.right)
            node.key = temp.key
            node.right = self._delete_recursive(node.right, temp.key)

        return node

    def _find_min(self, node):
        # 주어진 노드를 루트로 하는 서브트리에서 최소값을 찾음
        current = node
        while current.left:
            current = current.left
        return current
