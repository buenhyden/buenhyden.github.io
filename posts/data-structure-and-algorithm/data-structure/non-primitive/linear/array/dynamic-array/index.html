동적 배열 (Dynamic Array)#

동적 배열은 크기가 가변적인 배열 형태의 데이터 구조이다.
정적 배열과 달리 실행 시간에 크기를 조절할 수 있어 메모리를 효율적으로 사용할 수 있다.

특징#

  1. 가변적 크기: 필요에 따라 크기가 자동으로 조절된다.
  2. 연속된 메모리 할당: 요소들이 메모리상에 연속적으로 저장된다.
  3. 임의 접근(Random Access): 인덱스를 통해 O(1) 시간에 요소에 접근할 수 있다.
  4. 동적 메모리 할당: 실행 시간에 메모리를 할당하고 해제할 수 있다.

장점#

  1. 유연성: 크기를 미리 정하지 않아도 되어 유연한 데이터 관리가 가능하다.
  2. 메모리 효율성: 필요한 만큼만 메모리를 사용한다.
  3. 빠른 접근 속도: 인덱스를 통한 빠른 요소 접근이 가능하다.

단점#

  1. 재할당 비용: 크기 조정 시 새로운 메모리 할당과 데이터 복사에 따른 비용이 발생한다.
  2. 메모리 단편화: 빈번한 크기 조정으로 메모리 단편화가 발생할 수 있다.
  3. 삽입/삭제 연산의 비효율성: 중간에 요소를 삽입하거나 삭제할 때 다른 요소들을 이동시켜야 한다.

응용#

  1. 리스트 구현: Java의 ArrayList, Python의 list 등의 기본 자료구조 구현에 사용된다.
  2. 스택과 큐 구현: 동적 배열을 이용해 스택과 큐를 효율적으로 구현할 수 있다.
  3. 데이터베이스 인덱싱: 빠른 검색을 위한 인덱스 구조에 활용된다.

동작 원리#

  1. 초기화: 일정 크기의 배열로 시작한다.
  2. 삽입: 배열이 가득 차면 더 큰 크기의 새 배열을 생성하고 기존 데이터를 복사한다.
  3. 삭제: 요소를 제거하고 필요시 배열 크기를 줄인다.
  4. 접근: 인덱스를 통해 직접 접근한다.

구성 요소#

  1. 내부 배열: 실제 데이터를 저장하는 고정 크기 배열
  2. 크기(size): 현재 저장된 요소의 수
  3. 용량(capacity): 내부 배열의 전체 크기

public class DynamicArray<T> {
    private T[] array;
    private int size;
    private int capacity;

    @SuppressWarnings("unchecked")
    public DynamicArray(int initialCapacity) {
        array = (T[]) new Object[initialCapacity];
        size = 0;
        capacity = initialCapacity;
    }

    public void add(T element) {
        if (size == capacity) {
            grow();
        }
        array[size++] = element;
    }

    @SuppressWarnings("unchecked")
    private void grow() {
        capacity *= 2;
        T[] newArray = (T[]) new Object[capacity];
        System.arraycopy(array, 0, newArray, 0, size);
        array = newArray;
    }

    public T get(int index) {
        if (index < 0 || index >= size) {
            throw new IndexOutOfBoundsException();
        }
        return array[index];
    }

    public int size() {
        return size;
    }
}
