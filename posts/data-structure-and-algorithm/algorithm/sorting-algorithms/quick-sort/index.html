퀵 정렬 (Quick Sort)#

퀵 정렬(Quick Sort)은 분할 정복(divide and conquer) 전략을 사용하는 고성능 정렬 알고리즘이다. 1960년 Tony Hoare가 개발했으며, 평균적으로 **O(n log n)**의 시간 복잡도를 가지며 대규모 데이터셋 처리에 효율적이다.

퀵 정렬은 실시간 시스템과 고성능 컴퓨팅에서 널리 사용되며, 알고리즘 최적화의 핵심 사례로 연구된다.
피벗 선택 전략과 하이브리드 기법을 적용하면 현대 애플리케이션에서도 뛰어난 성능을 발휘한다.

기본 원리#

핵심 개념#

  • 피벗(Pivot) 선택: 배열에서 하나의 요소를 선택한다. 피벗 선택 방법은 성능에 큰 영향을 미칩니다(예: 첫 번째/중간/랜덤 요소 또는 중앙값).
  • 분할(Partitioning): 피벗을 기준으로 배열을 두 부분으로 나눈다. 피벗보다 작은 요소는 왼쪽, 큰 요소는 오른쪽으로 이동한다.
  • 재귀적 정렬: 분할된 서브 배열에 대해 동일한 과정을 재귀적으로 반복한다.

분할 과정 예시#

배열 [5, 3, 8, 4, 2]에서 피벗을 중간 값인 4로 선택:

  1. 피벗 4를 기준으로 분할:
    • 왼쪽: [3, 2] (4보다 작음)
    • 오른쪽: [5, 8] (4보다 큼)
  2. 재귀적으로 서브 배열 정렬:
    • [2, 3] → 정렬 완료
    • [5, 8] → 정렬 완료
  3. 최종 병합: [2, 3, 4, 5, 8]

시간 복잡도#

  • 평균 및 최선의 경우: O(n log n)
    • 피벗이 배열을 균등하게 분할할 때마다 재귀 깊이가 log n으로 유지된다.
  • 최악의 경우: O(n²)
    • 피벗이 항상 최소/최대값으로 선택되면 배열이 한쪽으로 치우쳐 재귀 깊이가 n이 된다(예: 이미 정렬된 배열).

공간 복잡도#

  • O(log n): 재귀 호출 스택의 깊이가 log n으로 제한된다.
  • 최악의 경우 O(n): 피벗 선택이 최악일 때 재귀 깊이가 n까지 증가한다.

장단점#

장점#

  • 속도: 평균적으로 가장 빠른 정렬 알고리즘 중 하나이다.
  • 제자리 정렬(In-place): 추가 메모리 없이 배열 내부에서 정렬된다.
  • 캐시 효율성: 참조 지역성(Locality of Reference)이 좋아 CPU 캐시 활용도가 높다.

단점#

  • 불안정 정렬: 동일 값의 요소가 원본 순서를 유지하지 않을 수 있다.
  • 피벗 선택 의존성: 최악의 경우 성능이 급격히 저하된다.
  • 재귀 오버헤드: 깊은 재귀 호출 시 스택 오버플로우 가능성이 있다.

최적화 전략#

  1. 랜덤 피벗 선택: 최악의 경우 확률을 낮춘다.
  2. 삼중 값 중앙값(Median-of-Three): 첫 번째, 중간, 마지막 요소 중 중앙값을 피벗으로 선택한다.
  3. 하이브리드 접근: 작은 서브 배열(예: 10개 미만)에 대해 삽입 정렬을 사용한다.
  4. 꼬리 재귀 제거: 재귀 대신 반복문을 사용해 스택 오버플로우를 방지한다.

실제 적용 사례#

  • 데이터베이스 정렬: 대용량 레코드의 빠른 정렬에 활용된다(예: Oracle, MySQL).
  • 게임 엔진: 실시간 렌더링을 위한 객체 Z-Order 정렬.
  • 머신러닝: 특성 선택 시 고차원 데이터 정렬에 사용된다.

def quick_sort(arr):
    # 배열의 길이가 1 이하면 이미 정렬된 상태
    if len(arr) <= 1:
        return arr
    
    # 피벗을 중간 값으로 선택 (다른 선택 방법도 가능)
    pivot = arr[len(arr) // 2]
    
    # 피벗을 기준으로 세 그룹으로 분할
    left = [x for x in arr if x < pivot]
    middle = [x for x in arr if x == pivot]
    right = [x for x in arr if x > pivot]
    
    # 재귀적으로 각 부분을 정렬하고 병합
    return quick_sort(left) + middle + quick_sort(right)

def optimized_quick_sort(arr, left=None, right=None):
    # 초기 호출시 인덱스 설정
    if left is None:
        left = 0
    if right is None:
        right = len(arr) - 1
    
    def partition(low, high):
        # 피벗 선택 (여기서는 마지막 원소를 피벗으로 사용)
        pivot = arr[high]
        i = low - 1  # 작은 원소들의 경계
        
        # 피벗보다 작은 원소들을 왼쪽으로 이동
        for j in range(low, high):
            if arr[j] <= pivot:
                i += 1
                arr[i], arr[j] = arr[j], arr[i]
        
        # 피벗을 올바른 위치로 이동
        arr[i + 1], arr[high] = arr[high], arr[i + 1]
        return i + 1
    
    if left < right:
        # 파티션을 나누고 피벗의 위치를 얻음
        pivot_idx = partition(left, right)
        
        # 피벗을 기준으로 양쪽을 재귀적으로 정렬
        optimized_quick_sort(arr, left, pivot_idx - 1)
        optimized_quick_sort(arr, pivot_idx + 1, right)
    
    return arr
