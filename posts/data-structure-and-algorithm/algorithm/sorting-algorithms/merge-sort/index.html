병합 정렬 (Merge Sort)#

병합 정렬(Merge Sort)은 분할 정복(divide and conquer) 방식을 사용하는 비교 기반 정렬 알고리즘이다.
존 폰 노이만이 1945년 개발한 이 알고리즘은 대규모 데이터셋 처리에 적합하며, **안정 정렬(Stable Sort)**의 특성을 가진다. 모든 경우(최선, 평균, 최악)에서 **O(n log n)**의 시간 복잡도를 보장한다.

병합 정렬은 데이터 무결성과 안정성이 중요한 시스템(예: 금융 거래 로그)에서 선호된다.
현대 프로그래밍 언어의 표준 라이브러리(예: Python sorted(), Java Collections.sort())에서도 하이브리드 방식으로 병합 정렬을 활용한다.

병합 정렬의 작동 원리#

  1. 분할(Divide):
    배열을 절반으로 나누는 과정을 재귀적으로 반복한다. 각 하위 배열의 길이가 1이 될 때까지 분할한다.

  2. 정복(Conquer):
    분할된 하위 배열을 재귀적으로 정렬한다. 길이가 1인 배열은 이미 정렬된 상태로 간주한다.

  3. 병합(Merge):
    정렬된 두 하위 배열을 순차적으로 비교하며 하나의 정렬된 배열로 합친다.

병합 정렬의 동작 과정#

예를 들어, [38, 27, 43, 3, 9, 82, 10]이라는 리스트를 오름차순으로 정렬하는 과정을 살펴보자:

  1. 분할: 리스트를 절반으로 나눈다:
    • [38, 27, 43, 3]
    • [9, 82, 10]
  2. 재귀적 분할: 각 하위 리스트를 다시 절반으로 나눈다:
    • [38, 27] → [38], [27]
    • [43, 3] → [43], [3]
    • [9],[82, 10] → [82], [10]
  3. 정복: 하위 리스트의 원소가 하나가 될 때까지 분할한 후, 이제 병합하면서 정렬한다:
    • [38],[27] → [27, 38]
    • [43], [3] → [3, 43]
    • [82], [10] → [10, 82]
  4. 병합: 정렬된 하위 리스트를 병합한다:
    • [27, 38], [3, 43] → [3, 27, 38, 43]
    • [9], [10, 82] → [9, 10, 82]
  5. 최종 병합: 모든 정렬된 하위 리스트를 병합하여 최종 정렬된 리스트를 얻는다:
    • [3, 27, 38, 43], [9, 10, 82] → [3, 9, 10, 27, 38, 43, 82]

def merge_sort(arr):
    if len(arr) <= 1:
        return arr
    
    mid = len(arr) // 2
    left = merge_sort(arr[:mid])  # 왼쪽 분할
    right = merge_sort(arr[mid:])  # 오른쪽 분할
    
    return merge(left, right)

def merge(left, right):
    result = []
    i = j = 0
    
    while i < len(left) and j < len(right):
        if left[i] <= right[j]:
            result.append(left[i])
            i += 1
        else:
            result.append(right[j])
            j += 1
    
    # 남은 요소 처리
    result.extend(left[i:])
    result.extend(right[j:])
    return result

# 예시 실행
arr = [4, 2, 6, 3, 7, 8, 5, 1]
print(merge_sort(arr))  # 출력: [1, 2, 3, 4, 5, 6, 7, 8]

시간 복잡도#

  • 모든 경우 O(n log n):
    • 분할 단계에서 log n 레벨이 생성되고, 각 레벨에서 n번의 비교/병합 연산이 수행된다.
      • 분할: 배열을 반으로 나누는 데 O(log n).
      • 병합: 각 레벨에서 O(n).
      • 총 시간: O(n log n).

공간 복잡도#

  • O(n):
    병합 과정에서 임시 배열이 필요하다. 원본 배열과 동일한 크기의 추가 메모리를 사용한다.

장단점#

장점#

  • 안정 정렬: 동일한 값의 상대적 순서가 유지된다.
  • 대규모 데이터 처리: 외부 정렬(디스크 기반)에 적합하다.
  • 일관된 성능: 입력 데이터의 초기 상태와 무관하게 **O(n log n)**을 보장한다.

단점#

  • 추가 메모리 필요: 임시 배열 사용으로 메모리 사용량이 높다.
  • 작은 데이터셋 비효율: 재귀 오버헤드로 인해 삽입/버블 정렬보다 느릴 수 있다.

실제 적용 사례#

  1. 외부 정렬(External Sorting):
    메모리보다 큰 데이터를 정렬할 때(예: 대용량 파일).
  2. 링크드 리스트 정렬:
    연결 리스트의 경우 추가 메모리 없이 O(1) 공간으로 구현 가능하다.
  3. 멀티코어 병렬 처리:
    분할 단계를 병렬화해 대용량 데이터를 빠르게 처리한다.

다른 정렬 알고리즘과 비교#

  • 퀵 정렬(Quick Sort):
    • 평균 **O(n log n)**이지만 최악 O(n²).
    • 캐시 효율성이 높으나 안정 정렬이 아니다.
  • 힙 정렬(Heap Sort):
    • 추가 메모리 없이 O(1) 공간 사용하지만 불안정 정렬.
