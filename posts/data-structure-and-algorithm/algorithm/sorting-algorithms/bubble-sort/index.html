버블 정렬 (Bubble Sort)#

버블 정렬(Bubble Sort)은 인접한 요소를 반복적으로 비교하고 교환하여 정렬하는 간단한 알고리즘이다.
정렬 과정에서 큰 값이 물속 거품처럼 위로 올라오는 모습을 닮아 이름이 붙었다. 작은 데이터 세트나 교육용으로 주로 활용되며, 구현이 쉽지만 성능이 낮은 특징을 가진다.

알고리즘 작동 원리#

  1. 비교: 배열의 첫 번째 요소부터 시작해 인접한 두 요소(arr[i]와 arr[i+1])를 비교한다.
  2. 교환: 순서가 잘못된 경우(arr[i] > arr[i+1]) 두 요소의 위치를 교환한다.
  3. 반복: 배열의 끝까지 이 과정을 반복하면 가장 큰 요소가 마지막 위치로 이동한다.
  4. 패스 완료: 한 번의 전체 순회(패스)를 마치면 정렬 범위를 하나 줄이고 과정을 반복한다.

예시: [5, 3, 8, 4, 2]

  • 1차 패스:
    5 ↔ 3 → [3, 5, 8, 4, 2]
    5 ↔ 8 → 유지
    8 ↔ 4 → [3, 5, 4, 8, 2]
    8 ↔ 2 → [3, 5, 4, 2, 8]
  • 2차 패스:
    3 ↔ 5 → 유지
    5 ↔ 4 → [3, 4, 5, 2, 8]
    5 ↔ 2 → [3, 4, 2, 5, 8]
  • 3차 패스:
    3 ↔ 4 → 유지
    4 ↔ 2 → [3, 2, 4, 5, 8]
  • 4차 패스:
    3 ↔ 2 → [2, 3, 4, 5, 8]

시간 복잡도#

  • 최선: O(n) (이미 정렬된 배열 + 최적화 적용 시)
  • 평균/최악: O(n²)
    • 비교 횟수: $$ \frac{n(n-1)}{2} $$
    • 교환 횟수: 최대 $$ \frac{n(n-1)}{2} $$

공간 복잡도#

  • O(1): 추가 메모리 없이 배열 내에서 교환만 수행(In-place 알고리즘).

장단점#

장점#

  • 구현이 매우 간단하여 알고리즘 학습용으로 적합하다.
  • 안정 정렬(Stable Sort): 동일 값의 요소가 원본 순서를 유지한다.
  • 작은 데이터 세트에서 효율적 (1,000개 미만).

단점#

  • 대규모 데이터에 비효율적: 시간 복잡도가 O(n²)로 성능이 급격히 저하된다.
  • 불필요한 교환 다수 발생: 이미 정렬된 부분도 계속 비교한다.

    def bubble_sort(arr):
    n = len(arr)
    for i in range(n):
        swapped = False
        for j in range(0, n-i-1):
            if arr[j] > arr[j+1]:
                arr[j], arr[j+1] = arr[j+1], arr[j]
                swapped = True
        if not swapped:  # 교환이 없으면 종료
            break
    return arr
    • 교환이 발생하지 않으면 정렬 완료로 간주하고 조기 종료한다.

사용 사례#

  • 교육 및 학습: 정렬 알고리즘의 기본 개념을 설명할 때.
  • 소규모 데이터: 데이터 양이 매우 적을 때.
  • 거의 정렬된 배열: 최적화 적용 시 빠르게 완료된다.
