그리디 알고리즘 (Greedy Algorithms)#

“탐욕적” 또는 “욕심쟁이” 알고리즘이라고도 불리며, 현재 상황에서 가장 최적의 선택을 하는 방식으로 문제를 해결하는 알고리즘.
매 선택의 순간마다 당장 눈앞에 보이는 최적의 선택을 하여 최종적인 해답에 도달하는 방식이다.

예를 들어, 거스름돈을 계산할 때 가장 큰 단위의 동전부터 사용하는 것이 그리디 알고리즘의 대표적인 예시이다.
500원짜리 동전을 최대한 사용하고, 그 다음 100원, 50원 순으로 사용하는 방식.

장점:

  1. 단순성: 직관적이고 이해하기 쉬워서 구현이 간단하다.
  2. 속도: 매 단계에서 최적의 선택을 하므로 계산 속도가 빠르다.
  3. 효율성: 많은 문제에서 적절한 해를 빠르게 제공한다.
  4. 근사해 제공: NP-Hard 문제에서 근사해를 빠르게 찾을 수 있다.

단점:

  1. 최적해 보장 불가: 항상 최적해를 보장하지 않는다.
  2. 문제 특성 의존성: 그리디 특성을 가진 문제에만 적용 가능하다.
  3. 전역 최적화 부재: 지역적 최적해가 전역 최적해를 보장하지 않는다.

특징#

  1. 탐욕적 선택 속성(Greedy Choice Property): 각 단계에서 현재 상태에서 최적의 선택을 한다.
  2. 최적 부분 구조(Optimal Substructure): 문제의 최적해가 부분 문제의 최적해로 구성될 수 있다.
  3. 단계적 선택: 문제를 해결하는 과정에서 매 단계마다 가장 최적의 선택을 한다.
  4. 비가역적 선택: 한번 선택한 것은 다시 번복하지 않는다.

그리디 알고리즘 종류와 특징#

동전 거스름돈 문제#

가장 큰 단위의 동전부터 사용

  • 장점: 간단하고 빠른 해결 가능
  • 단점: 특정 동전 체계에서만 최적해 보장
  • 활용 사례: 자판기, 화폐 교환 시스템

활동 선택 문제 (Activity Selection Problem)#

종료 시간을 기준으로 활동 선택

  • 장점: 최대 활동 수를 효율적으로 선택
  • 단점: 활동의 중요도를 고려하지 않음
  • 활용 사례: 회의실 예약 시스템, 작업 스케줄링

분할 가능 배낭 문제 (Fractional Knapsack Problem)#

단위 무게당 가치를 기준으로 물건 선택

  • 장점: 최적해 보장
  • 단점: 물건을 분할할 수 있는 경우에만 적용 가능
  • 활용 사례: 자원 할당, 투자 포트폴리오 구성

크러스컬 알고리즘 (Kruskal’s Algorithm)#

최소 신장 트리를 찾는 알고리즘

  • 장점: 희소 그래프에서 효율적
  • 단점: 밀집 그래프에서는 프림 알고리즘보다 성능이 떨어짐
  • 활용: 네트워크 설계, 통신망 구축

허프만 코딩(Huffman Coding)#

빈도수에 따라 가변 길이 코드 할당

  • 장점: 효율적인 데이터 압축 가능
  • 단점: 빈도수 정보가 필요함
  • 활용 사례: 데이터 압축, 파일 압축 알고리즘

다익스트라 알고리즘(Dijkstra’s Algorithm)#

최단 경로를 찾기 위해 가장 가까운 노드 선택

  • 장점: 단일 출발점 최단 경로 문제 해결에 효율적
  • 단점: 음의 가중치를 가진 간선에 적용 불가
  • 활용 사례: 네트워크 라우팅, GPS 내비게이션 시스템

최신 트렌드와 발전 방향#

  1. 다중 목적 그리디 알고리즘: 여러 목적을 동시에 고려하는 알고리즘 개발
  2. 그리디 알고리즘과 기계학습의 결합: 복잡한 최적화 문제 해결에 활용
  3. 근사 알고리즘으로서의 활용: NP-Hard 문제의 근사해 도출에 사용

그리디 알고리즘 선택 시 고려사항#

  1. 문제의 최적 부분 구조 확인
  2. 탐욕적 선택 속성 만족 여부 검토
  3. 단계적 결정 가능성 평가
  4. 정렬이나 우선순위 큐 관련 문제인지 확인

활용 사례#

  1. 네트워크 프로토콜: Open Shortest Path First 프로토콜
  2. 인공지능: 실시간 얼굴 추적
  3. 데이터 압축: 허프만 인코딩
  4. 그래프 알고리즘: 최소 신장 트리, 최단 경로 문제
  5. 스케줄링: 작업 스케줄링, 회의실 배정
  6. 리소스 할당: 분할 가능 배낭 문제
  7. 금융: 포트폴리오 최적화
