7:T4321,<h2 id="1-overview">1. Overview</h2>
<p><code>컴퓨터 과학 및 공학(Computer Science &#x26; Engineering, CS&#x26;E)</code>은 이 지식 저장소의 **최상위 루트 노드(Pillar Root)**입니다. 본 문서는 분산된 기술 지식을 유기적으로 연결하는 지식의 원천(SSOT, Single Source of Truth) 역할을 수행하며, 전산학의 이론적 기초와 공학적 실무 적용 사이의 균형 잡힌 로드맵을 제공합니다.</p>
<p>CS&#x26;E는 단순히 코드를 작성하는 기술을 넘어, 계산의 본질적 한계를 이해하고 복잡한 시스템의 신뢰성과 효율성을 보장하는 학문적/기술적 체계를 구축하는 것을 목표로 합니다. 대학생과 비전공 개발자가 현대적 엔지니어링의 정수를 체계적으로 수취할 수 있도록 안내합니다.</p>
<h2 id="2-scope-boundaries">2. Scope &#x26; Boundaries</h2>
<h3 id="in-scope">In-Scope</h3>
<ul>
<li><strong>Academic Core</strong>: CS2023 기반의 17개 지식 영역(KAs) 중 핵심 주제의 매핑 및 분류.</li>
<li><strong>Engineering Practice</strong>: SWEBOK v4.0 및 SFIA v9 기반의 소프트웨어 수명 주기(SDLC) 및 산업 역량 통합.</li>
<li><strong>Specialized Domains</strong>: 보안(CyBOK), 데이터 과학(DS-BoK) 등 부문별 핵심 지식 체계의 전사적 적용.</li>
<li><strong>Taxonomy Governance</strong>: 4단계 계층 구조(Top > Mid > Cluster > +a)를 통한 지식의 구조화 및 일관성 관리.</li>
</ul>
<h3 id="out-of-scope">Out-of-Scope</h3>
<ul>
<li>특정 프레임워크나 언어의 버전별 API 사용법과 같은 휘발성 도구 튜토리얼.</li>
<li>특정 산업군에만 한정되는 협소한 비즈니스 도메인 지식.</li>
<li>보편적 원리가 아닌 특정 환경의 일시적 트러블슈팅 기록.</li>
</ul>
<h3 id="boundaries">Boundaries</h3>
<ul>
<li><strong>Root vs Pillar</strong>: 본 루트 문서는 **지식의 위치(Single Home)**를 결정하는 거버넌스 기준점이 됩니다. 각 기술 주제는 14개의 Top Category 중 단 하나에만 귀속되며, 영역 간 경합 시 CS2023의 정의를 우선하여 경계를 획정합니다.</li>
</ul>
<h2 id="3-counterexample">3. Counterexample</h2>
<ul>
<li><strong>단순 '코딩' 위주의 학습</strong>: 내부 작동 원리(OS, Architecture)나 이론적 분석(Data Structures, Algorithms) 없이 특정 언어의 문법만 습득하는 행위는 CS&#x26;E의 본질적 탐구 영역을 벗어납니다.</li>
<li><strong>파편화된 정보 조각 수집</strong>: 전체 시스템의 계층적 구조(Taxonomy)에 대한 이해 없이 검색 엔진의 단편적 해결책에만 의존하는 것은 엔지니어링 사고가 결여된 상태입니다.</li>
<li><strong>근거 없는 설계론</strong>: 공인된 지식 체계(Primary Evidence)를 무시하고 개인의 한정된 경험이나 검증되지 않은 블로그의 주장만을 바탕으로 아키텍처를 논하는 것은 지식의 정당성을 훼손합니다.</li>
</ul>
<h2 id="4-prerequisites">4. Prerequisites</h2>
<ul>
<li><strong>이산 구조 및 수학적 논리 (Basic)</strong>: 모든 컴퓨팅 사고와 알고리즘 설계의 기초가 되는 수리적 기반입니다. (P1)</li>
<li><strong>정보 모델링 및 추상화 기초 (Recommended)</strong>: 현실 세계의 복잡한 데이터를 수리적으로 모델링하고 처리하는 능력입니다. (P1, P4)</li>
<li><strong>분류학적 사고와 공학적 태도 (Practical)</strong>: 지식의 위계를 정의하고 근거 기반으로 의사결정하는 엔지니어링 정서입니다. (P5)</li>
</ul>

<h2 id="6-learning-topics">6. Learning Topics</h2>
<h3 id="basic">Basic</h3>
<h4 id="core-topic-01-cse-taxonomy">Core Topic 01: 현대 전산학의 정의와 분류학 (CS&#x26;E Taxonomy)</h4>
<ul>
<li><strong>Why to Learn</strong>: 파편화된 기술 스택을 학문의 정석과 공학적 실무 관점으로 구조화하기 위함입니다.</li>
<li><strong>What to Learn</strong>:
<ul>
<li>CS2023의 17개 Knowledge Areas와 이 저장소의 14개 Pillars 매핑 규칙</li>
<li>SWEBOK 기반 소프트웨어 수명 주기(SDLC)의 물리적 선후 관계</li>
<li>SFIA v9 기반 엔지니어링 기술 성숙도 측정표</li>
</ul>
</li>
<li><strong>How to Learn</strong>:
<ul>
<li><code>posts/_index.md</code>에서 정의한 14개 Top Category의 수평적/수직적 연결 관계 확인</li>
<li>본인의 현재 지식 수준을 14개 축 위에 수치적으로 매핑</li>
</ul>
</li>
<li><strong>Implement</strong>: 14개 핵심 축(Pillars) 기반의 개인화된 전문성 레이더 차트</li>
</ul>
<h3 id="recommended">Recommended</h3>
<h4 id="core-topic-02-ssot-mechanics">Core Topic 02: 지식 거버넌사와 중복 제거 (SSOT Mechanics)</h4>
<ul>
<li><strong>Why to Learn</strong>: 대규모 지식 시스템에서 정보 유실과 중복으로 인한 엔트로피 증가를 방지하기 위함입니다.</li>
<li><strong>What to Learn</strong>:
<ul>
<li>Single Source of Truth (SSOT) 및 Single Home 관리 원칙</li>
<li>상위 노드의 Boundary 정의를 통한 하위 노드 책임 격리 기법</li>
<li>'See Also' 및 Aliases를 활용한 크로스 레퍼런스 거버넌스</li>
</ul>
</li>
<li><strong>How to Learn</strong>:
<ul>
<li>실제 하위 노드(<code>_index.md</code>)를 작성하며 상위 노드와 중복되는 설명을 제거하는 실습</li>
<li>변경된 slug와 url이 기존 링크를 깨뜨리지 않도록 Redirection 논리 검토</li>
</ul>
</li>
<li><strong>Implement</strong>: 주제별 Canonical 경로를 정의한 전사적 JSON Route Map</li>
</ul>
<h3 id="practical">Practical</h3>
<h4 id="core-topic-03-evidence-based-logic">Core Topic 03: 근거 기반 엔지니어링 (Evidence-based Logic)</h4>
<ul>
<li><strong>Why to Learn</strong>: 주관적 취향이 아닌 검증된 학문적/업계 표준(BoK)을 바탕으로 기술적 의사결정을 내리기 위해서입니다.</li>
<li><strong>What to Learn</strong>:
<ul>
<li>Primary Evidence Set(P1~P5)의 인용 및 적용 가이드라인</li>
<li>Terminology 테이블의 Evidence 칼럼과 실제 BoK 본문 연결 기법</li>
<li>9개 섹션 표준 템플릿의 수리적 충실도 검증</li>
</ul>
</li>
<li><strong>How to Learn</strong>:
<ul>
<li>ADR(Architecture Decision Record) 작성 시 BoK의 특정 챕터를 인용하여 근거 강화</li>
<li>기존 문서 속에 숨어있는 개인적 편견(Bias)을 식별하고 BoK 정의로 보정</li>
</ul>
</li>
<li><strong>Implement</strong>: 특정 기술 아키텍처 선택에 대한 BoK 기반 Rationale 보고서</li>
</ul>
<h3 id="advanced">Advanced</h3>
<h4 id="core-topic-04-emerging-tech-fusion">Core Topic 04: 신기술 통합 및 시스템 확장 (Emerging Tech Fusion)</h4>
<ul>
<li><strong>Why to Learn</strong>: 급변하는 기술 트렌드(LLM, Web3, Quantum 등)를 기초 학문 체계와 충돌 없이 융합하기 위해서입니다.</li>
<li><strong>What to Learn</strong>:
<ul>
<li>Emerging Core 기술의 수리적 본질 분석 및 기존 14개 축 배치 로직</li>
<li>새로운 분야의 지식 체계(BoK) 발굴 및 기존 ET(Evidence Trace)와 통합</li>
<li>시스템 확장 시 발생하는 도메인 간 교차 영향도 평가 기술</li>
</ul>
</li>
<li><strong>How to Learn</strong>:
<ul>
<li>BoK에 명시되지 않은 최신 논문을 읽고 기존 CS&#x26;E Taxonomy 내 "Single Home"을 논리적으로 할당</li>
<li>신기술 도입에 따른 성능/보안/비용 트레이드오프 수치화</li>
</ul>
</li>
<li><strong>Implement</strong>: 신기술 채택 시 기존 시스템 무결성 유지 및 회귀 방지 거버넌스 가이드</li>
</ul>

## Security Features

The button shortcode automatically handles security for external links:

- **External links** get `rel="noreferrer"` attribute
- **External links opening in new tab** get `rel="noopener noreferrer"`
- **Internal links opening in new tab** get `rel="noopener"`

This ensures safe navigation while maintaining functionality.

- Proper `role="button"` attribute
- `aria-label` support
- Keyboard focus indicators
- High contrast focus rings
- Disabled state handling
