블룸 필터 (Bloom filter)#

블룸 필터(Bloom Filter)는 공간 효율적인 확률적 데이터 구조로, 원소가 집합에 속하는지 여부를 빠르게 확인하는 데 사용된다.
1970년 Burton Howard Bloom이 고안한 이 구조는 **거짓 양성(false positive)**은 허용하지만 **거짓 음성(false negative)**은 절대 발생하지 않는다.

블룸 필터는 빠른 검색과 극도의 공간 효율이 필요한 시스템에서 필수적이다.
특히 대용량 데이터 처리, 실시간 애플리케이션, 메모리 제약 환경에서 강력한 성능을 발휘한다.
다만 정확성이 절대적이라면 전통적 해시 테이블이 더 적합하다.

핵심 구성 요소#

  • 비트 배열(Bit Array): 모든 비트가 0으로 초기화된 배열 (크기 m)
  • 해시 함수(Hash Functions): 원소를 비트 배열의 인덱스로 매핑하는 k개의 독립적 해시 함수

동작 과정#

  1. 삽입(Add)
    원소를 k개의 해시 함수로 해싱 → 각 결과값을 비트 배열의 인덱스로 사용 → 해당 위치의 비트를 1로 설정.
    예시: 원소 “apple"을 3개의 해시 함수로 해싱 → 인덱스 1, 4, 7 → 비트 배열 [0,1,0,1,0,0,1,0,0,0] 갱신.

  2. 조회(Contains)
    원소를 같은 k개 해시 함수로 해싱 → 모든 인덱스의 비트가 1인지 확인.

    • 1개라도 0: 원소가 집합에 없음 (100% 확실)
    • 모두 1: 원소가 집합에 있을 수 있음 (일정 확률로 거짓 양성)

수학적 분석#

거짓 양성 확률#

  • 공식: $$ P = \left(1 - e^{-kn/m}\right)^k $$
    • m: 비트 배열 크기
    • k: 해시 함수 개수
    • n: 저장된 원소 수
  • 최적화: $$ k = \frac{m}{n} \ln 2 $$

일 때 최소 오류율 달성

공간 효율성#

  • 기존 해시 테이블 대비 약 10배 적은 메모리 사용 (예: 1억 개 원소 저장 시 약 114MB)
  • 실제 데이터 저장 없이 존재 여부만 추적하므로 공간 절약

주요 특징#

  • 공간 효율성: 블룸 필터는 비트 배열과 해시 함수를 사용하여 데이터를 저장하므로, 동일한 작업을 수행하는 다른 데이터 구조에 비해 메모리 사용량이 적다.
  • 빠른 삽입 및 조회: 데이터의 삽입과 조회 시 여러 해시 함수를 사용하여 비트 배열의 특정 위치를 설정하거나 확인하므로, 시간 복잡도가 O(k)로 매우 빠르다.
  • 거짓 양성 허용: 블룸 필터는 거짓 양성을 허용하므로, 요소가 실제로는 집합에 속하지 않지만 속하는 것으로 판단될 수 있습니다. 그러나 거짓 음성은 발생하지 않는다.

장단점 비교#

장점단점
- 초고속 검색(O(1))- 원소 삭제 불가
- 메모리 효율적(9.6비트/원소)- 거짓 양성 발생
- 대규모 데이터 처리 적합- 해시 충돌 관리 필요
- 분산 시스템에 용이- 실제 데이터 저장 불가

실제 응용 사례#

  1. 데이터베이스 최적화
    • Cassandra, HBase: 디스크 조회 전 블룸 필터로 존재 여부 확인
  2. 콘텐츠 전송 네트워크(CDN)
    • Akamai: 자주 요청되지 않는 콘텐츠(One-hit wonders) 캐싱 방지
  3. 보안 시스템
    • Chrome 브라우저: 악성 URL 1차 필터링
  4. 분산 처리
    • MapReduce: 대용량 데이터셋 효율적 처리

변형 및 확장#

  • 카운팅 블룸 필터(Counting Bloom Filter): 비트 대신 카운터 사용 → 삭제 지원
  • 스케일러블 블룸 필터(Scalable Bloom Filter): 동적 확장 가능
  • 공간 효율성 개선: Quotient Filter, Cuckoo Filter

    class DuplicateChecker:
    def __init__(self, expected_items=1000000):
        size = optimal_bloom_filter_size(expected_items, 0.01)
        num_hash = optimal_number_of_hash_functions(expected_items, size)
        self.bloom_filter = BloomFilter(size, num_hash)

    def is_duplicate(self, item):
        if self.bloom_filter.contains(item):
            return True  # 아마도 중복
        self.bloom_filter.add(item)
        return False  # 확실히 새로운 항목

    class DatabaseCache:
    def __init__(self):
        self.bloom_filter = BloomFilter(1000000, 7)
        self.cache = {}

    def query(self, key):
        # 블룸 필터로 먼저 검사하여 불필요한 디스크 접근 방지
        if not self.bloom_filter.contains(key):
            return None  # 확실히 없음
        return self.cache.get(key)  # 캐시에서 검색

    def should_allow_packet(self, ip):

import hashlib
import math

class BloomFilter:
    def __init__(self, size, hash_count):
        self.size = math.ceil(size * 1.2)  # 20% 여유 공간
        self.hash_count = hash_count
        self.bit_array = [0] * self.size

    def add(self, element):
        for seed in range(self.hash_count):
            index = self._hash(element, seed) % self.size
            self.bit_array[index] = 1

    def contains(self, element):
        for seed in range(self.hash_count):
            index = self._hash(element, seed) % self.size
            if self.bit_array[index] == 0:
                return False
        return True

    def _hash(self, element, seed):
        return int(hashlib.sha1(f"{element}{seed}".encode()).hexdigest(), 16)
