Message Brokers#

애플리케이션 간 통신을 중재하여 상호 의존성을 최소화한다.
Message Broker는 현대적인 분산 시스템과 마이크로서비스 아키텍처에서 중요한 역할을 하며, 시스템 간 효율적인 통신과 데이터 교환을 가능하게 한다.

메시지 브로커의 기초 개념#

메시지 브로커란?#

메시지 브로커는 애플리케이션, 시스템 및 서비스가 서로 통신할 수 있도록 하는 중간 프로그램이다. 발신자와 수신자 사이의 중개자 역할을 하여 메시지의 검증, 저장, 라우팅 및 전달을 담당한다.

메시지 브로커의 주요 기능#

  • 비동기 통신: 발신자와 수신자가 동시에 활성화될 필요가 없다.
  • 메시지 큐잉: 처리 대기 중인 메시지를 저장한다.
  • 메시지 라우팅: 메시지를 적절한 수신자에게 전달한다.
  • 메시지 변환: 다양한 시스템 간 데이터 형식을 변환한다.
  • 신뢰성 보장: 메시지 전달 및 처리를 보장한다.
  • 부하 분산: 여러 소비자 간에 작업을 분산한다.

주요 메시지 교환 패턴#

  1. 점대점(Point-to-Point): 하나의 발신자가 하나의 수신자에게 메시지를 보낸다. 각 메시지는 정확히 한 번만 처리된다.
  2. 발행-구독(Publish-Subscribe): 발신자(게시자)가 메시지를 특정 주제(토픽)에 게시하면, 해당 주제를 구독한 모든 수신자가 메시지를 수신한다.
  3. 요청-응답(Request-Reply): 클라이언트가 요청을 보내고 서버가 응답하는 패턴이다.
  4. 경쟁 소비자(Competing Consumers): 여러 소비자가 동일한 큐에서 메시지를 소비하지만, 각 메시지는 한 소비자에게만 전달된다.

주요 메시지 브로커 시스템#

Apache Kafka#

고성능, 고가용성의 분산 이벤트 스트리밍 플랫폼이다.

주요 특징:

  • 높은 처리량과 낮은 지연 시간
  • 내구성과 확장성이 뛰어남
  • 데이터 스트림 처리 지원
  • 분산 아키텍처
  • 고가용성

사용 사례:

  • 실시간 데이터 파이프라인
  • 로그 집계
  • 스트림 처리
  • 이벤트 소싱

2.2 RabbitMQ#

오픈 소스 메시지 브로커로, AMQP(Advanced Message Queuing Protocol)를 구현한다.

주요 특징:

  • 다양한 메시징 프로토콜 지원
  • 유연한 라우팅 기능
  • 클러스터링과 고가용성
  • 관리 UI 제공
  • 플러그인 시스템

사용 사례:

  • 마이크로서비스 간 통신
  • 백그라운드 작업 처리
  • 비동기 처리

Redis#

인메모리 데이터 구조 저장소로, 메시지 브로커로도 활용된다.

주요 특징:

  • 높은 성능
  • 간단한 구현
  • 다양한 데이터 구조 지원
  • 퍼시스턴스 옵션

사용 사례:

  • 간단한 메시징 요구사항
  • 실시간 채팅
  • 작업 큐

ActiveMQ#

아파치의 오픈 소스 메시지 브로커로, JMS API를 지원한다.

주요 특징:

  • JMS 1.1 및 2.0 지원
  • 다양한 크로스 언어 클라이언트
  • 여러 전송 프로토콜 지원
  • 관리 콘솔 제공

사용 사례:

  • 엔터프라이즈 통합
  • Java 기반 시스템

Amazon SQS/SNS#

AWS의 관리형 메시지 큐 서비스.

주요 특징:

  • 서버리스로 확장성이 높음
  • 관리 오버헤드가 낮음
  • AWS 서비스와의 통합

사용 사례:

  • AWS 기반 마이크로서비스
  • 클라우드 네이티브 애플리케이션

// build.gradle에 의존성 추가: implementation 'org.apache.kafka:kafka-clients:3.1.0'

import org.apache.kafka.clients.producer.*;
import java.util.Properties;

public class KafkaProducerExample {
    public static void main(String[] args) {
        // 생산자 설정
        Properties props = new Properties();
        props.put("bootstrap.servers", "localhost:9092");
        props.put("key.serializer", "org.apache.kafka.common.serialization.StringSerializer");
        props.put("value.serializer", "org.apache.kafka.common.serialization.StringSerializer");
        
        Producer<String, String> producer = new KafkaProducer<>(props);
        
        // 메시지 보내기
        ProducerRecord<String, String> record = 
            new ProducerRecord<>("my-topic", "key", "Hello Kafka");
            
        producer.send(record, new Callback() {
            @Override
            public void onCompletion(RecordMetadata metadata, Exception e) {
                if (e != null) {
                    e.printStackTrace();
                } else {
                    System.out.println("Message sent to partition " + metadata.partition() +
                                      " with offset " + metadata.offset());
                }
            }
        });
        
        producer.flush();
        producer.close();
    }
}

// build.gradle에 의존성 추가: implementation 'org.apache.kafka:kafka-clients:3.1.0'

import org.apache.kafka.clients.consumer.*;
import java.time.Duration;
import java.util.Collections;
import java.util.Properties;

public class KafkaConsumerExample {
    public static void main(String[] args) {
        // 소비자 설정
        Properties props = new Properties();
        props.put("bootstrap.servers", "localhost:9092");
        props.put("group.id", "my-group");
        props.put("key.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");
        props.put("value.deserializer", "org.apache.kafka.common.serialization.StringDeserializer");
        props.put("auto.offset.reset", "earliest");
        
        Consumer<String, String> consumer = new KafkaConsumer<>(props);
        
        // 토픽 구독
        consumer.subscribe(Collections.singletonList("my-topic"));
        
        // 메시지 소비
        try {
            while (true) {
                ConsumerRecords<String, String> records = consumer.poll(Duration.ofMillis(100));
                for (ConsumerRecord<String, String> record : records) {
                    System.out.printf("Received message: key = %s, value = %s, partition = %d, offset = %d%n",
                                     record.key(), record.value(), record.partition(), record.offset());
                }
            }
        } finally {
            consumer.close();
        }
    }
}

실무 구현 - 심화#

메시지 브로커 아키텍처 설계 고려사항#

  1. 고가용성 설계

    • 클러스터링: 여러 노드로 구성된 클러스터를 구성하여 단일 장애점 제거
    • 복제: 메시지 데이터 복제로 데이터 손실 방지
    • 자동 장애 조치: 한 노드가 실패할 경우 자동으로 다른 노드로 전환

  2. 확장성 전략

    • 수평적 확장: 노드 추가를 통한 처리량 증가
    • 파티셔닝: 데이터를 여러 파티션으로 분할하여 병렬 처리
    • 샤딩: 부하를 여러 브로커에 분산

  3. 메시지 내구성

    • 디스크 저장: 메시지를 디스크에 저장하여 브로커 장애 시에도 데이터 보존
    • 확인 메커니즘: 소비자가 메시지 처리를 확인하는 메커니즘 구현
    • 재시도 로직: 실패한 메시지 처리를 위한 재시도 전략

// build.gradle에 의존성 추가: implementation 'org.apache.kafka:kafka-streams:3.1.0'

import org.apache.kafka.common.serialization.Serdes;
import org.apache.kafka.streams.KafkaStreams;
import org.apache.kafka.streams.StreamsBuilder;
import org.apache.kafka.streams.StreamsConfig;
import org.apache.kafka.streams.kstream.KStream;
import org.apache.kafka.streams.kstream.KTable;
import org.apache.kafka.streams.kstream.Produced;

import java.util.Arrays;
import java.util.Properties;

public class WordCountExample {
    public static void main(String[] args) {
        Properties props = new Properties();
        props.put(StreamsConfig.APPLICATION_ID_CONFIG, "wordcount-app");
        props.put(StreamsConfig.BOOTSTRAP_SERVERS_CONFIG, "localhost:9092");
        props.put(StreamsConfig.DEFAULT_KEY_SERDE_CLASS_CONFIG, Serdes.String().getClass());
        props.put(StreamsConfig.DEFAULT_VALUE_SERDE_CLASS_CONFIG, Serdes.String().getClass());

        StreamsBuilder builder = new StreamsBuilder();
        
        // 입력 토픽에서 텍스트 라인 읽기
        KStream<String, String> textLines = builder.stream("text-lines");
        
        // 단어 분리 및 카운트
        KTable<String, Long> wordCounts = textLines
            .flatMapValues(value -> Arrays.asList(value.toLowerCase().split("\\W+")))
            .groupBy((key, word) -> word)
            .count();
            
        // 결과를 출력 토픽으로 전송
        wordCounts.toStream()
            .to("word-count", Produced.with(Serdes.String(), Serdes.Long()));
            
        KafkaStreams streams = new KafkaStreams(builder.build(), props);
        streams.start();
        
        // 종료 시 리소스 해제
        Runtime.getRuntime().addShutdownHook(new Thread(streams::close));
    }
}

import org.springframework.amqp.core.*;
import org.springframework.amqp.rabbit.connection.ConnectionFactory;
import org.springframework.amqp.rabbit.core.RabbitTemplate;
import org.springframework.amqp.support.converter.Jackson2JsonMessageConverter;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;

@Configuration
public class RabbitMQConfig {

    public static final String QUEUE_NAME = "my-queue";
    public static final String EXCHANGE_NAME = "my-exchange";
    public static final String ROUTING_KEY = "my-routing-key";

    @Bean
    Queue queue() {
        return QueueBuilder.durable(QUEUE_NAME)
                .withArgument("x-dead-letter-exchange", "dlx")
                .withArgument("x-dead-letter-routing-key", "dlq")
                .build();
    }

    @Bean
    DirectExchange exchange() {
        return new DirectExchange(EXCHANGE_NAME);
    }

    @Bean
    Binding binding(Queue queue, DirectExchange exchange) {
        return BindingBuilder.bind(queue).to(exchange).with(ROUTING_KEY);
    }
    
    // 데드 레터 큐 설정
    @Bean
    Queue deadLetterQueue() {
        return QueueBuilder.durable("dlq").build();
    }
    
    @Bean
    DirectExchange deadLetterExchange() {
        return new DirectExchange("dlx");
    }
    
    @Bean
    Binding deadLetterBinding() {
        return BindingBuilder.bind(deadLetterQueue()).to(deadLetterExchange()).with("dlq");
    }

    @Bean
    public Jackson2JsonMessageConverter converter() {
        return new Jackson2JsonMessageConverter();
    }

    @Bean
    public RabbitTemplate rabbitTemplate(ConnectionFactory connectionFactory) {
        RabbitTemplate rabbitTemplate = new RabbitTemplate(connectionFactory);
        rabbitTemplate.setMessageConverter(converter());
        return rabbitTemplate;
    }
}

import org.springframework.amqp.rabbit.core.RabbitTemplate;
import org.springframework.beans.factory.annotation.Autowired;
import org.springframework.stereotype.Service;

@Service
public class MessageProducer {

    private final RabbitTemplate rabbitTemplate;

    @Autowired
    public MessageProducer(RabbitTemplate rabbitTemplate) {
        this.rabbitTemplate = rabbitTemplate;
    }

    public void sendMessage(Object message) {
        rabbitTemplate.convertAndSend(
            RabbitMQConfig.EXCHANGE_NAME, 
            RabbitMQConfig.ROUTING_KEY, 
            message
        );
    }
}

메시지 브로커 성능 최적화#

  1. RabbitMQ 성능 튜닝

    • 메모리 설정: vm_memory_high_watermark 조정
    • 디스크 I/O 최적화: 빠른 스토리지 사용, 디스크 폴링 간격 조정
    • 클러스터 크기 조정: 적절한 노드 수 유지
    • 메시지 크기: 큰 메시지는 분할하거나 참조로 대체
    • 프리페치 카운트: 소비자의 메시지 버퍼링 조정

  2. Kafka 성능 튜닝

    • 파티션 수: 토픽당 적절한 파티션 수 결정
    • 배치 크기 조정: 생산자 배치 설정
    • 압축 사용: 네트워크 및 스토리지 효율성을 위한 메시지 압축
    • JVM 설정: 힙 크기 및 GC 설정 최적화
    • 디스크 I/O: 다중 디스크 사용, RAID 구성
