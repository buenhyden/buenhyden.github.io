Caddy#

Caddy는 Go 언어로 작성된 현대적이고 강력한 오픈 소스 웹 서버이다.

주요 특징#

  1. 자동 HTTPS: Let’s Encrypt를 통해 SSL/TLS 인증서를 자동으로 획득 및 갱신
  2. 간단한 설정: 직관적인 Caddyfile을 통한 쉬운 구성
  3. HTTP/2 및 HTTP/3 지원
  4. 리버스 프록시 및 로드 밸런싱 기능
  5. 정적 파일 서빙에 최적화

설치 및 기본 사용법#

  1. 설치:
    Ubuntu/Debian:

2

4

    sudo apt install -y debian-keyring debian-archive-keyring apt-transport-https
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/gpg.key' | sudo gpg --dearmor -o /usr/share/keyrings/caddy-stable-archive-keyring.gpg
curl -1sLf 'https://dl.cloudsmith.io/public/caddy/stable/debian.deb.txt' | sudo tee /etc/apt/sources.list.d/caddy-stable.list
sudo apt update
sudo apt install caddy

  2. 기본 설정:
    /etc/caddy/Caddyfile 파일을 편집하여 설정한다.

  3. 간단한 Caddyfile 예시:

    root * /var/www/example.com
    file_server
}

  4. Caddy 시작:

    1

    sudo systemctl start caddy

  5. 상태 확인:

    1

    sudo systemctl status caddy

example.com {
    root * /var/www/example.com
    file_server
    encode gzip
    log {
        output file /var/log/caddy/access.log
    }
}

# 리버스 프록시 설정
api.example.com {
    reverse_proxy localhost:3000
}

# PHP 애플리케이션 설정
php.example.com {
    root * /var/www/php-app
    php_fastcgi unix//run/php/php8.0-fpm.sock
    file_server
}

SSL/TLS 자동 설정#

Caddy의 가장 큰 특징 중 하나는 자동 HTTPS 설정이다.
별도의 설정 없이도 도메인에 대한 SSL/TLS 인증서를 자동으로 발급하고 관리한다:

example.com {
    # HTTPS가 자동으로 설정됨
    root * /var/www/example.com
    file_server
    
    # 추가적인 TLS 설정 (선택사항)
    tls {
        protocols tls1.3
        curves x25519
        alpn h2 http/1.1
    }
}

# 로드 밸런싱 설정
balance.example.com {
    reverse_proxy {
        to 10.0.0.1:80 10.0.0.2:80 10.0.0.3:80
        lb_policy round_robin
        health_check /health
        health_check_interval 10s
    }
}

# URL 리다이렉션
example.com {
    redir https://www.example.com{uri} permanent
}

# 정적 파일 압축
static.example.com {
    root * /var/www/static
    file_server
    encode gzip zstd
    header /* {
        Cache-Control "public, max-age=31536000"
    }
}
