Serverless Architecture

서버 인프라 관리 없이 애플리케이션을 개발하고 실행할 수 있는 클라우드 컴퓨팅 실행 모델

Serverless Architecture#

서버리스 아키텍처는 개발자가 서버 인프라를 관리하지 않고도 애플리케이션을 구축하고 실행할 수 있게 해주는 클라우드 컴퓨팅 모델입니다. 이 모델에서는 클라우드 제공업체가 서버 관리와 용량 계획을 담당하며, 코드 실행에 필요한 인프라를 동적으로 할당합니다.

서버리스 아키텍처의 핵심 구성 요소#

1. 함수형 컴퓨팅 (FaaS - Function as a Service)#

FaaS는 서버리스 아키텍처의 핵심 요소로, 개발자가 개별 함수를 작성하고 이벤트에 응답하여 실행되도록 할 수 있습니다. 대표적인 FaaS 플랫폼으로는 AWS Lambda, Azure Functions, Google Cloud Functions 등이 있습니다.

주요 특징:

  • 이벤트 기반 실행
  • 자동 확장 및 축소
  • 마이크로빌링 (사용한 만큼만 지불)
  • 스테이트리스(Stateless) 함수

2. 백엔드 서비스 (BaaS - Backend as a Service)#

BaaS는 개발자가 직접 구현하지 않고도 사용할 수 있는 관리형 백엔드 서비스를 제공합니다. 인증, 데이터베이스, 스토리지, API 게이트웨이 등이 포함됩니다.

주요 서비스:

  • 관리형 데이터베이스 (DynamoDB, Firestore)
  • 인증 서비스 (Cognito, Firebase Auth)
  • 스토리지 서비스 (S3, Cloud Storage)
  • API 게이트웨이

서버리스 아키텍처의 장단점#

장점#

  1. 운영 복잡성 감소: 서버 프로비저닝, 패치, 유지보수 등의 인프라 관리 부담 제거
  2. 비용 효율성: 실제 사용량에 따른 요금 부과 (유휴 리소스에 대한 비용 없음)
  3. 자동 확장성: 트래픽 증가에 따라 자동으로 확장되며 별도의 구성 필요 없음
  4. 개발 속도 향상: 인프라 구성보다 비즈니스 로직 구현에 집중 가능
  5. 높은 가용성: 클라우드 제공업체가 고가용성과 내결함성 보장

단점#

  1. 콜드 스타트 지연: 비활성 함수의 첫 실행 시 초기화 지연 발생
  2. 벤더 종속성: 특정 클라우드 제공업체의 서비스에 의존하게 됨
  3. 로컬 테스트의 어려움: 클라우드 환경을 로컬에서 정확히 시뮬레이션하기 어려움
  4. 장기 실행 작업에 부적합: 대부분의 FaaS 플랫폼은 실행 시간 제한이 있음
  5. 모니터링 및 디버깅 복잡성: 분산 환경에서 문제 추적이 어려울 수 있음

서버리스 설계 패턴#

1. 이벤트 처리 패턴#

특정 이벤트(HTTP 요청, 파일 업로드, 데이터베이스 변경 등)에 응답하여 함수가 실행됩니다.

2. 팬아웃 패턴#

하나의 이벤트가 여러 병렬 함수를 트리거하여 작업을 병렬로 처리합니다.

3. 단계적 워크플로 패턴#

여러 함수가 순차적으로 실행되어 복잡한 비즈니스 프로세스를 처리합니다.

4. 분기 및 조정 패턴#

여러 함수의 결과를 하나로 통합하여 최종 결과를 생성합니다.

서버리스 구현 고려사항#

  1. 함수 세분화 수준: 함수를 얼마나 작게 나눌 것인지 결정
  2. 상태 관리: 스테이트리스 환경에서 상태 정보 관리 방법
  3. 인증 및 권한 부여: 보안 구현 전략
  4. API 설계: 효율적인 API 게이트웨이 구성
  5. 데이터 지속성: 적절한 데이터 저장소 선택
  6. 성능 최적화: 콜드 스타트 최소화 및 함수 성능 향상 방법

주요 서버리스 플랫폼#

  1. AWS Serverless: Lambda, API Gateway, DynamoDB, S3, SNS, SQS, Step Functions
  2. Azure Serverless: Azure Functions, Logic Apps, Event Grid, Cosmos DB
  3. Google Cloud Serverless: Cloud Functions, Cloud Run, Firebase, Firestore
  4. 기타 플랫폼: Vercel, Netlify Functions, Cloudflare Workers

서버리스 사용 사례#

  1. 웹 API 및 마이크로서비스: REST 또는 GraphQL API 구현
  2. 실시간 파일 처리: 이미지 리사이징, 문서 변환, 미디어 인코딩
  3. 데이터 처리 파이프라인: ETL 작업, 데이터 변환, 분석
  4. IoT 백엔드: 센서 데이터 수집 및 처리
  5. 챗봇 및 가상 비서: 자연어 처리 및 대화형 인터페이스
  6. 스케줄된 작업: 정기적인 백업, 보고서 생성, 알림

이 섹션에서는 서버리스 아키텍처의 개념, 구현 방법, 모범 사례, 실제 사용 사례, 그리고 주요 클라우드 제공업체의 서버리스 서비스에 대해 상세히 다룹니다.
