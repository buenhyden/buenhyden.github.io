Allocation Policy#

캐시 정책에서 할당 정책(allocation policy)은 캐시 미스가 발생했을 때 데이터를 어떻게 처리할지를 결정하는 중요한 요소이다.
할당 정책은 주로 두 가지 유형으로 나뉜다:
쓰기 할당(write-allocate)과 비쓰기 할당(no-write-allocate)이다.

캐시 할당 정책은 시스템의 성능과 효율성에 큰 영향을 미치므로, 응용 프로그램의 특성과 요구사항을 고려하여 신중하게 선택해야 한다. 동적 할당 정책과 같은 최신 기술을 활용하면 더욱 최적화된 캐시 관리가 가능할 것이다.

  1. 쓰기 할당 (Write-Allocate)
    쓰기 할당 정책은 캐시 미스 시 해당 메모리 위치에 대한 캐시 블록을 할당한 후 데이터를 캐시에 쓰는 방식이다.
    장점:
    - 후속 읽기 작업의 성능 향상
    - 쓰기 후 읽기 작업이 빈번한 경우 효율적
    단점:
    - 캐시 공간을 더 많이 사용
    - 초기 쓰기 작업이 더 오래 걸릴 수 있음

  2. 비쓰기 할당 (No-Write-Allocate)
    비쓰기 할당 정책은 캐시 미스 시 데이터를 직접 주 메모리에 쓰고 캐시 블록을 할당하지 않는 방식이다.
    장점:
    - 캐시 오염 감소
    - 쓰기 작업만 수행하는 경우 효율적
    단점:
    - 후속 읽기 작업 시 캐시 미스 가능성 증가
    - 주 메모리 접근 빈도 증가

할당 정책의 조합#

할당 정책은 일반적으로 쓰기 정책(write-through 또는 write-back)과 함께 사용된다.

  1. 쓰기 백(Write-Back) + 쓰기 할당(Write-Allocate):

    • 가장 일반적인 조합
    • 쓰기 성능이 중요하고 읽기 후 쓰기 작업이 빈번한 경우 적합

  2. 쓰기 관통(Write-Through) + 비쓰기 할당(No-Write-Allocate):

    • 데이터 일관성이 중요하고 쓰기 후 즉시 읽기가 드문 경우 적합

동적 할당 정책#

최근 연구에서는 동적 캐시 할당 정책의 효과성이 입증되고 있다.
이 접근 방식은 시스템의 요구사항에 따라 실시간으로 캐시 할당을 조정한다.
장점:

  • 크리티컬 태스크의 예측 가능성 향상
  • 비크리티컬 태스크의 성능 저하 완화
    적용 예:
  • 실시간 시스템에서 100Hz 정도의 제어 주파수로 동작 가능
  • 크리티컬 태스크 실행 중에만 캐시 메모리 할당, 종료 시 해제

할당 정책 선택 시 고려사항#

  1. 쓰기 빈도: 쓰기가 빈번한 경우 쓰기 할당이 유리할 수 있다.
  2. 읽기 빈도: 읽기가 빈번한 경우도 쓰기 할당이 유리할 수 있다.
  3. 데이터 일관성 요구사항: 일관성이 중요한 경우 쓰기 관통과 비쓰기 할당의 조합을 고려할 수 있다.
  4. 시스템 성능 목표: 전반적인 시스템 효율성과 읽기/쓰기 성능의 균형을 고려해야 한다.
  5. 워크로드 특성: 데이터 접근 패턴에 따라 적절한 정책을 선택해야 한다.
