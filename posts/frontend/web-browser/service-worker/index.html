Service Worker#

Service Worker는 웹 애플리케이션의 성능과 사용자 경험을 혁신적으로 개선하는 백그라운드 스크립트 기술이다. 브라우저와 네트워크 사이에서 프록시 역할을 하며, 오프라인 지원, 캐싱, 푸시 알림 등을 가능하게 한다.

Service Worker는 오프라인 경험과 앱 같은 성능을 웹에 도입하는 핵심 기술이다. 캐싱 전략과 생명주기 관리가 성공적 구현의 열쇠이며, PWA 개발 시 필수적으로 활용된다.

정의와 핵심 역할#

  • 프록시 서버: 웹 앱과 서버 사이에서 네트워크 요청을 가로채고 조작한다.
  • 오프라인 지원: 캐시된 리소스 제공으로 인터넷 연결 없이도 앱 사용이 가능하다.
  • 독립 실행 환경: 메인 스레드와 분리되어 브라우저 종료 후에도 백그라운드에서 작동한다.

필수 보안 조건#

  • HTTPS 필수: 외부 환경에서는 HTTPS 프로토콜에서만 동작 (로컬호스트 예외).
  • 스코프 제한: 서비스 워커 파일 경로에 따라 제어 범위 결정 (예: /sw.js → 루트 및 하위 경로).

서비스 워커의 제한 사항#

  • DOM 접근 불가: 서비스 워커는 워커 컨텍스트에서 실행되므로 DOM에 직접 접근할 수 없다. 따라서 UI를 직접 조작하는 작업은 수행할 수 없다.
  • HTTPS 필요: 보안상의 이유로, 서비스 워커는 HTTPS 프로토콜을 통해 제공되는 페이지에서만 동작한다. 이는 네트워크 요청을 가로채는 특성상 중간자 공격에 취약할 수 있기 때문이다.

    // 캐시 우선 전략 구현
self.addEventListener('fetch', event => {
    event.respondWith(
        caches.match(event.request)
            .then(response => {
                if (response) {
                    return response;
                }

                return fetch(event.request)
                    .then(response => {
                        // 응답 복제 후 캐시 저장
                        if (response.status === 200) {
                            const responseClone = response.clone();
                            caches.open('dynamic-cache')
                                .then(cache => {
                                    cache.put(event.request, responseClone);
                                });
                        }
                        return response;
                    })
                    .catch(() => {
                        // 오프라인일 때 폴백 페이지 제공
                        return caches.match('/offline.html');
                    });
            })
    );
});

    // 리소스 유형에 따른 다른 캐싱 전략 적용
self.addEventListener('fetch', event => {
    const url = new URL(event.request.url);

    if (url.pathname.startsWith('/api/')) {
        // API 요청은 네트워크 우선
        event.respondWith(networkFirst(event.request));
    } else if (url.pathname.startsWith('/static/')) {
        // 정적 리소스는 캐시 우선
        event.respondWith(cacheFirst(event.request));
    }
});

    event.waitUntil(
        caches.open(PRECACHE)
            .then(cache => cache.addAll(PRECACHE_URLS))
            .then(self.skipWaiting())
    );
});

주요 주의사항#

  • 캐시 버전 관리: 업데이트 시 이전 캐시 반드시 삭제해야 함
  • 브라우저 지원 확인: IE 미지원, Chrome/Firefox/Safari 최신 버전 필요
  • 스코프 관리: 서비스 워커 파일 위치에 따라 제어 영역이 결정됨
