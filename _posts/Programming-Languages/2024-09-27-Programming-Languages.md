## 인터프리터 언어와 컴파일 언어
- 코드 실행 방식에 따라 구분된다. 
### 인터프리터 언어
-  코드를 한 줄씩 읽고, 해석하며, 실행하는 방식으로 작동한다. 
- 특징
	- 코드 실행시 해석이 이뤄진다
	-  디버깅이 용이하다
	-  플랫폼 독립성이 높다
- 장점
	- 개발 속도가 빠르며, 즉시 결과를 확인할 수 있다.
	- 오류를 쉽게 찾아 수정할 수 있다.
- 단점
	- 실행 속도가 느리다.
	- 배포시 인터프리터가 필요하다

> 인터프리터
> - 인터프리터 언어의 소스 코드를 해석하고 실행하는 프로그램.
> - 역할
>     - 코드 해석
>         - 인터프리터는 작성된 소스 코드를 한 줄씩 읽고, 이를 이해할 수 있는 형태로 변환
>         - 즉각적으로 실행 가능한 중간 코드나 기계어로 변환하여 실행
>     - 실행
> 	    - 해석된 코드를 즉시 실행한다. 이 과정에서 각 명령이 순차적으로 처리된다.
>     - 디버깅 지원
> 	    - 인터프리터는 실행 중 오류를 발경하고, 오류가 발생한 위치와 원인을 알려준다.
> 	    - 이는 개발자가 코드를 수정하고 테스트하는데 유용하다
>     - 메모리 관리
> 	    - 변수와 데이터 구조의 메모리를 동적으로 할당하고 해제한다.
> 	    - 필요에 따라 가비지 컬렉션을 통해 사용하지 않는 메모리를 정리한다.
>     - 플랫폼 독립성 제공
> 	    - 인터프리터가 설치된 환경에서 코드를 실행할 수 있도록 하여, 다양한 운영 체제에서 호환성을 제공한다.
> - 작동 과정
> 	1. **소스 코드 입력** : 사용자 또는 개발자가 작성한 소스 코드가 인터프리터에 입력된다.
> 	2. **구문 분석** : 인터프리터는 코드를 구문 분석하여 문법적 오류를 검사한다.
> 	3. **해석 및 실행** : 코드가 해석되어 명령이 실행된다. 이 과정에서 변수 값이 계산되고, 조건문 및 반복문이 처리된다.
> 	4. *결과 출력** : 실행 결과가 화면에 출력되거나, 파일에 저장된다.
### 컴파일러 언어
- 소스 코드를 전체적으로 읽고 기계어로 변환한 후 실행하는 언어
- 특징
	- 코드가 먼저 컴파일되어 실행 파일이 생성된다.
	- 최적화가 가능하다
- 장점
	- 실행 속도가 빠르다
	- 오류가 컴파일시 발견되므로, 런타임 오류가 줄어든다.
- 단점
	- 컴파일 시간이 필요하며, 개발 속도가 느려질 수 있다.
	- 플랫폼 종속적일 수 있으며, 다른 플랫폼에서 실행하려면 다시 컴파일해야 한다.

> 컴파일러
> - 컴파일러 언어의 소스 코드를 기계어 또는 중간 코드로 변환하는 프로그램
> - 역할
> 	- 소스 코드 변환
> 		- 작성된 소스 코드를 전체적으로 읽어, 이를 기계어 또는 중간 코드로 변환한다. 이 과정에서 원래 코드의 의미를 유지하면서 컴퓨터가 이해할 수 있는 형태로 변환한다.
> 	- 구문 분석
> 		- 소스 코드의 문법을 검사하여 구문 오류를 찾아낸다. 이를 통해 코드가 문법적으로 올바른지 확인한다.
> 	- 최적화
> 		- 코드의 성능 향상시키기 위해 최적화 과정을 거친다. 이는 실행 속도를 높이거나 메모리 사용을 줄이는 등의 방법을 포함한다.
> 	- 오류 검사
> 		- 컴파일 과정에서 발견된 오류를 사용자에게 보고한다. 이러한 오류는 문법 오류, 타입 오류 등 다양한 형태일 수 있다.
> 	- 목표 코드 생성
> 		- 변환된 기계어 코드 또는 중간 코드를 생성하여 실행 파일을 만든다. 이 파일은 독립적으로 실행할 수 있다.
> - 작동과정
> 	1. 소스코드 입력 : 사용자 또는 개발자가 작성한 소스 코드가 컴파일러에게 입력된다.
> 	2. 구문 분석 : 코드를 구문 분석하여 문법적 오류를 검사한다.
> 	3. 중간 코드 생성 : 소스 코드를 중간 코드로 변환한다. 이는 최적화와 코드 생성을 위한 중간 단계이다. 
> 	4. 최적화 : 중간 코드를 최적화하여 성능을 개선한다.
> 	5. 기계어 코드 생성 : 최종적으로 기계어 코드로 변환하여 실행 파일을 생성한다.
> 	6. 오류 보고 : 코드에 오류가 있을 경우, 해당 오류를 사용자에게 보고한다.

## 저수준 언어(Low Level Language)와 고수준 언어(High Level Language)
- 추상화 수준과 기계와의 상호 작용 방식에서 큰 차이를 보인다.
### 저수준 언어(Low Level Language)
- **기계 친화적** : 기계어 또는 어셈블리어와 같이 하드웨어에 가까운 언어로 CPU 명령어와 직접적으로 상호작용할 수 있어, 메모리와 프로세서의 세부 사항을 제어할 수 있음.
- **고성능** : 하드웨어와 가까운 수준에서 작동하므로 최적화된 성능을 제공할 수 있음.
- **복잡성** : 코드가 복잡하고 일기 어려워, 개발자가 실수를 하기 쉽다. 디버깅과 유지보수가 어렵다.
### 고수준 언어(High Level Language)
- **인간 친화적** : 자연어에 가까운 문법을 사용하여 개발자가 이해하고 작성하기 쉽게 설계되었다.
- **추상화** : 하드웨어 세부 사항을 추상화하여 개발자가 논리적 문제 해결에 집중할 수 있게 함.
- **생산성** : 코드 작성이 저수준 언어보다 빠르고 효율적이며, 유지보수가 용이하다.
- **이식성** : 플랫폼에 독립적이어서 여러 운영체제에서 실행될 수 있다.
