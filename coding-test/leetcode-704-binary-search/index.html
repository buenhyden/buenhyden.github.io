문제 설명 - Easy#

Given an array of integers nums which is sorted in ascending order, and an integer target, write a function to search target in nums. If target exists, then return its index. Otherwise, return -1.

You must write an algorithm with O(log n) runtime complexity.

Example 1:

Input: nums = [-1,0,3,5,9,12], target = 9
Output: 4
Explanation: 9 exists in nums and its index is 4

Example 2:

Input: nums = [-1,0,3,5,9,12], target = 2
Output: -1
Explanation: 2 does not exist in nums so return -1

Constraints:

  • 1 <= nums.length <= 104
  • -104 < nums[i], target < 104
  • All the integers in nums are unique.
  • nums is sorted in ascending order.

class Solution:

    def search(self, nums: List[int], target: int) -> int:

        if not 1 <= len(nums) <= 10^4:

            raise ValueError 

        left, right = 0, len(nums) -1
        while left <= right:
            mid = (left+right) // 2
            if nums[mid] == target:
                return mid
            elif nums[mid] < target:
                left = mid+1
            else:
                right = mid-1
        return -1

function binarySearch(nums, target) {
    let left = 0;
    let right = nums.length - 1;

    while (left <= right) {
        let mid = Math.floor((left + right) / 2);
        if (nums[mid] === target) {
            return mid;
        } else if (nums[mid] < target) {
            left = mid + 1;
        } else {
            right = mid - 1;
        }
    }
    return -1;
}

해설#

이분 탐색은 정렬된 배열에서 효율적으로 값을 찾는 알고리즘입니다. 핵심 아이디어는 배열의 중간값과 목표값(target)을 비교하며 탐색 범위를 점차 줄여나가는 것입니다. 주어진 배열이 정렬되어 있으면 이분 탐색은 매 단계에서 검색 범위를 반으로 줄여 O(log n)의 시간 복잡도로 값을 찾을 수 있습니다.

접근 방식:

  1. 배열의 시작점과 끝점을 나타내는 left와 right 변수를 설정합니다.
  2. while left <= right 조건을 유지하면서 탐색을 진행합니다.
  3. 각 반복마다 중간 지점(mid)을 계산하고, 중간 값과 target을 비교합니다:
    • mid값이 target보다 작다면 left를 mid + 1로 이동합니다.
    • mid값이 target보다 크다면 right를 mid - 1로 이동합니다.
    • mid값이 target과 같다면 해당 인덱스를 반환합니다.
  4. 반복문이 종료되면 값이 없는 경우이므로 -1을 반환합니다.

활용 팁:

  • 반드시 입력 데이터가 정렬되어 있어야 합니다.
  • 중복 값이 있는 경우에도 특정 위치를 찾는 데 유용합니다.
  • 문제에 따라 약간의 조건 수정으로 다른 유형의 검색 문제를 풀 수 있습니다.
