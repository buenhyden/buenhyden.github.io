생성 개발 완료 후 develop에 병합 네이밍: feature/기능명 release: 출시 준비 develop에서 분기 버그 수정, 문서 작업 등 릴리즈 준비 작업만 허용 완료시 master(main)와 develop에 병합 네이밍: release-버전 hotfix: 긴급 버그 수정 master(main)에서 분기 수정 후 master(main)와
