Fiber

Fiber (Lightweight Thread) Fiber는 경량 스레드(Lightweight Thread)로 알려져 있으며, 운영 체제의 커널 수준이 아닌 사용자 공간에서 관리되는 실행 단위를 의미한다. 이는 전통적인 스레드와 비교하여 더 적은 리소스를 사용하며, 프로그래머가 직접 제어할 수 있는 협력적 멀티태스킹(Cooperative Multitasking)을 구현하는 데 사용된다. Fiber의 정의와 특징 Fiber는 단일 스레드 내에서 여러 실행 흐름을 가능하게 하는 경량 프리미티브이다. 일반적인 스레드와 달리 협력적 멀티태스킹 방식을 사용한다. Fiber는 자체적으로 실행 권한을 양도하며, 스케줄러에 의해 강제로 선점되지 않는다. Fiber Vs 스레드 스레드는 선점형 멀티태스킹을 사용하지만, Fiber는 협력적 멀티태스킹을 사용합니다[4][6]. Fiber는 스레드보다 더 가볍고 메모리 사용량이 적어 대량 생성이 가능합니다[2]. 하나의 OS 스레드 위에 여러 Fiber가 존재할 수 있습니다[4]. Fiber의 장점 컨텍스트 스위칭 비용이 낮아 효율적인 동시성 처리가 가능합니다[2]. I/O 작업이 빈번한 상황에서 특히 유용합니다[2]. 스레드보다 더 많은 동시성 단위를 생성할 수 있어 확장성이 높습니다[4]. Fiber의 구현과 사용 Python에서는 fibers 라이브러리를 통해 Fiber를 구현할 수 있습니다[1][3]. Fiber 생성 시 target 함수와 인자를 지정하고, switch() 메서드로 실행을 제어합니다[1][3]. 1 2 3 4 5 6 7 import fibers def func(): print("Fiber 실행") f = fibers.Fiber(target=func) f.switch() Fiber와 코루틴의 관계 Fiber와 코루틴은 매우 유사한 개념입니다[6]. 코루틴은 주로 프로그래밍 언어 수준에서 구현되는 반면, Fiber는 시스템 수준에서 구현됩니다[6]. 주의사항 Fiber는 협력적 멀티태스킹을 사용하므로, 한 Fiber가 제어권을 양도하지 않으면 다른 Fiber가 실행될 수 없습니다[2][4]. 디버깅이 어려울 수 있으며, 잘못 사용 시 전체 시스템의 성능에 영향을 줄 수 있습니다[2]. Temporal Dead Zone
